# 2 Sigma 学习系统（2sigma-learn）

基于 Benjamin Bloom 的 2 Sigma 研究：**一对一辅导 + 掌握学习法** = 超过 98% 同龄人的学习效果。

核心机制：AI 主动发起诊断性提问，学习者作答，AI 据此校准下一份材料难度。文件系统保存进度，跨会话不丢失。

---

## 触发方式

```
/2sigma-learn [可选：主题名称]
```

自然语言触发：
- 我要学 [主题]
- 帮我系统学习 [主题]
- 开启/续接 [主题] 的学习
- 继续学习

---

## 执行流程

### Phase 0：侦测模式（新开 or 续接）

读取用户指定的学习目录（或询问），判断：

- **目录不存在 / 无文件** → 进入 Phase 1（新开）
- **目录已有文件** → 进入 Phase 2（续接）

---

### Phase 1：新开学习会话

**步骤 1：确认主题与起点**

如果用户未说明主题，问：
> 你想学什么主题？用一句话描述你目前对它的了解程度。

收到回答后，**不要追问更多**，直接进入步骤 2。

**步骤 2：建立文件结构**

```
mkdir ~/.Codex/learning/[主题名]/
```

**步骤 3：生成第一份学习文件**

文件路径：`~/.Codex/learning/[主题]/01-诊断与基础.md`

文件结构模板：
```markdown
# [主题] — 第 01 课：[本课核心概念]

> 难度校准：入门诊断（根据你的作答将调整后续难度）
> 生成时间：[日期]

---

## 内容（300-500字）

[适配用户声明水平的讲解。如未知水平，从最基础开始。
重点：用具体例子、类比，避免堆砌术语。]

---

## 诊断问题

请回答以下 3 个问题。不必完美，说出你目前的理解即可。

**Q1（记忆层）**：[能直接从内容中找到答案的基础问题]

**Q2（理解层）**：[需要用自己的话解释或举例的问题]

**Q3（应用层）**：[需要将概念应用到新情境的问题]

---
> 回答完毕后告诉我，我会生成下一份文件并说明难度调整依据。
```

生成文件后，告知用户：
```
已创建学习文件：~/.Codex/learning/[主题]/01-诊断与基础.md

请阅读内容并回答末尾的 3 个问题。
回答不必完整，说出你目前的真实理解就好——这是校准难度用的，不是考试。
```

---

### Phase 2：续接已有学习会话

**步骤 1：读取历史文件**

```
ls ~/.Codex/learning/[主题]/
```

找到最新的文件，检查末尾是否有未回答的诊断问题。

**步骤 2：展示当前状态**

```
已找到你的 [主题] 学习记录：
- 已完成：[N] 份文件
- 最新文件：[文件名]
- 当前难度：[评估描述]
- 待回答问题：[是/否]
```

**步骤 3：根据状态决定动作**

- 若有未回答问题 → 提示用户先作答
- 若已作答但未生成下一份 → 分析回答，生成下一份
- 若用户要求直接继续 → 生成下一份（难度维持）

---

### Phase 3：作答分析与难度校准

收到用户的问题回答后，执行：

**分析框架：**

| 评估维度 | 判断标准 |
|---------|---------|
| 核心概念掌握 | 能否准确描述，有无关键偏差 |
| 理解深度 | 是否只是复述，还是真正消化 |
| 应用能力 | 能否迁移到新情境 |

**校准规则：**

```
答对/基本正确 ≥ 2/3  →  下一份：难度 +1 档，引入新概念
答对/基本正确 = 1/3  →  下一份：难度持平，换角度重新讲解
答对/基本正确 = 0/3  →  下一份：难度 -1 档，填补前置漏洞
```

**生成下一份文件前，先输出校准报告：**

```
**作答分析：**
- ✅ 掌握：[概念A]——你的理解：[引用用户原话]
- ⚠️ 偏差：[概念B]——你说：[引用]，实际：[简短纠正]
- ❌ 盲点：[概念C]——需要补充

**难度决策：** [+1/-1/持平]，理由：[一句话]

正在生成第 [N] 份文件…
```

然后生成文件，路径：`~/.Codex/learning/[主题]/[序号]-[核心概念].md`

---

### Phase 4：阶段复盘（每 5 份文件触发一次）

当完成第 5、10、15… 份文件后，自动提议：

```
你已完成 [N] 份学习文件。建议做一次阶段复盘，生成：
1. 本阶段知识图谱（核心概念关系）
2. 你的典型误解模式（3 条）
3. 综合检验问题（3 个）
4. 下阶段学习路径建议

生成复盘文件？（直接说"是"或"继续学"跳过）
```

复盘文件路径：`~/.Codex/learning/[主题]/review-[序号]-阶段复盘.md`

---

## 文件命名规范

```
~/.Codex/learning/
└── [主题名]/
    ├── 01-诊断与基础.md
    ├── 02-[核心概念].md
    ├── 03-[核心概念].md
    ├── ...
    └── review-05-阶段复盘.md
```

序号始终两位数（01, 02 … 10, 11）。

---

## 禁止行为

- 不在用户未作答时自动生成下一份文件
- 不一次输出超过 600 字的内容
- 不跳过诊断问题
- 不询问多余的信息（奥卡姆剃刀：够用就好）
- 不主动解释这个 skill 的工作流程（做就是了）
