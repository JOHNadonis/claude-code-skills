# A股新闻资金雷达

把现有 `finance-news-radar` 新闻系统、MX 金融数据 skills、A股产业链映射、量化筛选和缠论择时组合起来，输出“新闻是否被资金确认，以及候选股是否具备可执行买卖点”的本地化投资雷达。

核心定位：先用新闻资金共振筛候选，再用量化和缠论做精筛与择时。默认输出条件化交易计划、失效条件和风控，不输出无条件买入指令。

## 数据调用规则

金融问题必须调用 MX skills 或其脚本，不能凭记忆编行情数据。

### 优先组合

| 需求 | 使用 |
|---|---|
| 结构化行情、涨跌幅、资金流、技术指标 | `mx-finance-data` |
| 板块/个股筛选、涨幅榜、资金流入榜 | `mx-stocks-screener` |
| 最新新闻、公告、研报、政策 | `mx-finance-search` |
| 宏观 CPI/PMI/利率/商品等 | `mx-macro-data` |

### 查询原则

1. 先读/刷新本地新闻数据，再查资金。默认正式运行必须先刷新新闻；只有用户明确说“用缓存”“不刷新”“只读现有 JSON”时才跳过刷新。
2. 先查板块，再查龙头个股。
3. 单次 `mx-finance-data` 查询最多 5 个实体，超过就分批。
4. MX 查询报错时先看 raw/描述文件，不要直接判定失败。
5. 输出必须标注数据时间和不确定性。

## 工作流

### 阶段一：刷新并读取真实新闻源

1. 检查项目目录是否存在。
2. 默认先执行本 skill 自带的新闻刷新脚本，输出到 skill 自己的 `data/` 目录：

```bash
python3 /Users/whc/.claude/skills/a-share-news-radar/scripts/update_finance.py
python3 /Users/whc/.claude/skills/a-share-news-radar/scripts/generate_summary.py --no-llm
```

3. 只有用户明确说“用缓存”“不刷新”“只读现有 JSON”时，才跳过上一步。
4. 刷新后读取：
   - `data/latest-24h.json`
   - `data/market-summary.json`
   - 可选：`data/sentiment.json`、`data/economic-calendar.json`
5. 检查 `generated_at`：
   - 若数据不是当天生成，必须明确警告“新闻数据可能过期”。
   - 若刷新脚本失败，先报告失败原因，再询问是否改用缓存，不要静默使用旧数据。
6. 从 `items_finance` 或 `items` 中提取过去 24 小时的金融新闻。
7. 不要使用记忆替代当前文件内容。

若用户给了特定主题或文章，优先分析用户给定主题；否则从 `latest-24h.json` 中抽取 A股相关主题。

### 阶段二：抽取主题和映射对象

对每条候选新闻提取：

```text
标题
来源
发布时间
市场标签
关键词
涉及公司
涉及产业链
映射A股板块
映射A股个股
情绪：利好 / 利空 / 中性 / 不确定
```

优先识别这些跨市场映射：

| 海外/新闻锚点 | A股映射方向 |
|---|---|
| NVDA / Blackwell / AI服务器 | AI算力、服务器、光模块、PCB、液冷 |
| LITE / COHR / AAOI / 光通信 | CPO、光模块、硅光、光芯片、高速PCB |
| AAPL / iPhone / Vision Pro | 苹果链、消费电子、PCB、声学、结构件 |
| TSLA / FSD / Robotaxi / Optimus | 新能源车、汽车零部件、机器人 |
| Micron / DRAM / NAND | 存储芯片、半导体周期 |
| Copper / Oil / Gold | 有色、油气、黄金、资源股 |
| 政策：低空/机器人/算力/半导体 | 对应政策主题板块 |

若映射不确定，标注“映射弱”，不要强行关联。

### 阶段三：查询 A股资金异动

按主题执行 MX 查询：

1. 板块层面：
   - 今日/近 3 日涨跌幅
   - 成交额
   - 主力净流入
   - 主力净流入占成交额比例
   - 领涨股/领跌股

2. 个股层面：
   - 涨跌幅
   - 成交额
   - 换手率
   - 主力净流入
   - 5 日 / 20 日涨幅
   - 是否突破近期高点

3. 可选加强：
   - 龙虎榜
   - 融资融券
   - 北向/大宗/机构行为
   - 最新公告/研报验证

推荐查询模板：

```text
mx-stocks-screener: 今日A股主力净流入最多的概念板块，涨跌幅、成交额、领涨股，select-type=板块
mx-finance-data: [板块/个股] 最近5个交易日涨跌幅、成交额、主力净流入、换手率
mx-finance-search: [主题/公司] 最新公告、研报、新闻、政策影响
```

### 阶段四：四象限分类

把每个主题归入一个象限：

| 象限 | 定义 | 含义 |
|---|---|---|
| 强共振 | 有新闻 + 有资金流入 + 龙头确认 | 优先关注 |
| 观察 | 有新闻 + 资金未确认 | 新闻热，市场未买账 |
| 潜伏异动 | 无明显新闻 + 有资金流入 | 可能有隐含消息或预期差 |
| 利好出货 | 有新闻 + 资金流出 / 高开低走 | 警惕兑现 |

判断时不要只看涨跌幅，必须同时看资金、成交额和龙头一致性。

### 阶段五：量化精筛

只对“强共振”和“潜伏异动”候选做量化精筛；“观察”和“利好出货”默认不进入买点分析，除非用户明确要求。

必查指标：

```text
趋势：5/10/20/60日均线、多头排列、价格相对20日线位置
动量：近5日/10日/20日涨幅、RSI、MACD、是否创新高
量能：成交额、量比、成交额相对20日均值放大倍数
资金：主力净流入、超大单净流入、连续净流入天数
波动：ATR或近20日振幅、是否高波动追涨区
相对强度：相对板块、相对沪深300/创业板指强弱
筹码风险：解禁、减持、股东户数、龙虎榜席位质量（可选）
```

量化评分 100 分：

| 维度 | 分数 | 规则 |
|---|---:|---|
| 趋势强度 | 0-20 | 均线多头、站上20日线、趋势未破 |
| 动量质量 | 0-20 | 突破、RSI不过热、MACD配合 |
| 量价配合 | 0-20 | 放量上涨、缩量回踩、无异常爆量滞涨 |
| 资金持续性 | 0-20 | 主力/超大单连续流入，流入占比合理 |
| 相对强度 | 0-10 | 强于板块和指数 |
| 风险扣分 | -20-0 | 高位长上影、放量滞涨、跳空过大、利好兑现 |

硬性过滤：

```text
跌破20日线且无快速收回 → 不给买点
连续大涨后放量长上影 → 不给追涨买点
主力净流出且股价高开低走 → 归入利好出货
板块强但个股弱于板块 → 降权
个股强但板块资金不确认 → 降权
```

### 阶段六：缠论结构择时

缠论只作为结构化择时工具，不替代资金和量化验证。若K线数据不足，明确说明“缠论结构无法确认”。

按以下顺序判断：

1. **级别选择**
   - 日线：判断趋势方向和中枢位置
   - 30分钟：判断买卖点精度
   - 5分钟：只用于盘中细化，不用于盘后强结论

2. **结构识别**
   - 是否存在明确上涨/下跌笔
   - 是否形成中枢
   - 当前价格在中枢上方、内部还是下方
   - 是否出现背驰或类背驰
   - 是否出现三类买卖点

3. **买点定义**
   - 一买：下跌趋势背驰后转折，风险高，需小仓验证
   - 二买：一买后回踩不破前低，资金重新确认
   - 三买：突破中枢后回踩不回中枢，趋势确认度最高

4. **卖点定义**
   - 一卖：上涨趋势背驰后转折
   - 二卖：反弹不创新高或弱反抽
   - 三卖：跌破中枢后反抽不上中枢

5. **优先级**
   - 新闻资金强共振 + 量化强 + 缠论三买：最高优先级
   - 新闻资金强共振 + 量化强 + 缠论二买：次优先级
   - 新闻资金强共振 + 量化强但无缠论买点：只观察，等结构
   - 有缠论买点但无资金确认：不作为高优先级

### 阶段七：精准买卖计划

输出必须是条件化计划，不是无条件荐股。

每只候选股输出：

```text
候选逻辑：新闻/板块/资金为什么选它
当前结构：趋势、量价、资金、缠论级别
理想买点：触发条件 + 价格区间
确认信号：放量突破 / 缩量回踩 / 主力净流入 / 板块同步
失效条件：跌破哪个位置、资金转负、板块走弱
止损位：结构止损 + 最大亏损比例
止盈位：第一目标、第二目标、移动止盈规则
仓位：试错仓 / 标准仓 / 加仓条件
等待条件：如果现在不是买点，等什么信号
```

仓位默认规则：

| 信号组合 | 仓位建议 |
|---|---:|
| 强共振 + 量化强 + 缠论三买 | 标准仓 50%-70% |
| 强共振 + 量化强 + 缠论二买 | 标准仓 30%-50% |
| 强共振 + 量化中 + 结构未确认 | 观察或试错仓 10%-20% |
| 有新闻无资金 / 利好出货 | 0% |

单票最大亏损默认不超过账户 1%-2%。若用户未给账户规模，只输出比例，不估算金额。

### 阶段八：雷达评分

默认 100 分制：

| 维度 | 分数 | 规则 |
|---|---:|---|
| 新闻热度 | 0-20 | 来源数量、重要性、是否政策/财报/产业催化 |
| 产业映射强度 | 0-15 | 映射是否纯，是否有明确A股链条 |
| 板块资金强度 | 0-25 | 主力净流入、成交额放大、资金连续性 |
| 龙头确认 | 0-20 | 龙头是否领涨、是否放量、是否突破 |
| 风险扣分 | -20-0 | 高开兑现、资金流出、退潮期、政策/财报不确定 |

等级：

```text
80-100：强共振
60-79：可观察
40-59：弱确认
0-39：噪音或反向信号
```

## 边界和风险

- 这不是投资建议。
- 不承诺胜率。
- 新闻和资金共振只代表市场关注度提高，不代表未来一定上涨。
- 高开过多、缩量、资金流出、退潮期都要降权。
- 没有数据时直接说“不知道”，不要编造。
