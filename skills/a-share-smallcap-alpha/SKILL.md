# A 股小盘高弹性研究

## 使用定位

这是一个研究工作流 skill，不直接给交易指令。目标是先排雷，再从小市值 A 股里寻找具备财务质量、产业故事、事件催化和交易活跃度的候选标的。

默认以中文回复。股票代码、字段名、命令、CSV/XLSX 文件路径保持英文或原始格式。

## 默认假设

用户没有给阈值时，使用以下默认值，并在输出中说明：

- 市场：沪深主板、创业板、科创板；北交所仅在用户要求或流动性达标时纳入。
- 小盘范围：总市值 20 亿至 200 亿元人民币；高风险弹性池可放宽到 10 亿至 300 亿元。
- 流动性：近 20 个交易日日均成交额不低于 5000 万元；北交所不低于 2000 万元。
- 排除：ST、*ST、退市整理、暂停上市、近一年被证监会立案调查、审计意见非标、重大资金占用、重大未决诉讼、控股股东高比例质押且无缓释方案。
- 财务底线：扣非净利润或经营现金流不能长期双弱；尚未盈利公司必须解释现金储备、融资能力和亏损收窄路径。
- 低价股：不要简单用“股价低于 5 元”作为价值信号；优先看市值、成交额、基本面和催化剂。

## 依赖的 MX skills

按任务需要加载或调用这些本机 skills：

- `MX_StockPick`：第一轮选股和板块/概念池生成。适合自然语言筛选 A 股、板块、指数成分、涨跌幅、市值、技术和主题条件。
- `MX_FinData`：对候选股补结构化指标。单次最多 5 个实体、3 个核心指标；超过限制时分批查询。
- `MX_FinSearch`：检索最新公告、新闻、研报和政策事件。用于验证催化剂、风险和市场忽视点。
- `MX_MacroData`：仅在需要宏观或商品价格背景时使用。调用前必须把地域、指标、品种和日期解包成明确值。

如果 `EM_API_KEY` 未配置，不能编造最新数据；改为输出可执行查询方案和不确定性。

## 标准流程

### 1. 明确任务类型

先判断用户要的是哪一种：

- **财务质量筛选**：寻找资产负债率低、现金充足、ROE 稳定、分红改善的小盘公司。
- **高弹性小盘机会**：寻找小市值、主题成长、近期催化、交易活跃的候选股。
- **低价/未盈利事件股**：寻找股价低、市值小、有重组/换帅/定增/机构注资等事件，但必须加强排雷。
- **单票分析**：围绕商业模式、行业地位、财务质量、估值、增长逻辑、风险、多空逻辑做结构化分析。

用户没给市场、行业、市值、时间窗口时，用默认假设继续，不要反复追问。

### 2. 生成初始股票池

优先用 `MX_StockPick`。查询语句要包含 A 股、小市值、非 ST、流动性、行业/主题、催化或财务条件。

示例：

```bash
cd /Users/whc/.codex/skills/MX_StockPick
python -m scripts.get_data --select-type A股 --query "A股总市值20亿到200亿元，非ST非退市整理，近20日日均成交额大于5000万元，所属AI算力、机器人、半导体设备、创新药、固态电池、低空经济、新能源出海概念，列出股票代码、股票名称、总市值、最新价、近20日成交额、行业、概念、涨跌幅、市盈率、市净率"
```

对“财务质量筛选”可改成：

```bash
cd /Users/whc/.codex/skills/MX_StockPick
python -m scripts.get_data --select-type A股 --query "A股总市值20亿到300亿元，非ST非退市整理，资产负债率低于30%，流动比率大于2，近三年ROE高于15%，连续三年现金分红，近20日日均成交额大于5000万元，列出股票代码、名称、总市值、行业、资产负债率、流动比率、ROE、股息率、成交额"
```

### 3. 排雷过滤

用 `MX_FinSearch` 检索每只候选股的公告和风险。优先查最近 6 至 12 个月。

必须检查：

- ST、*ST、退市风险、交易所问询函、监管函、立案调查。
- 审计意见、资金占用、违规担保、重大诉讼。
- 控股股东质押、减持、解禁、定增失败。
- 重组是否真实推进，还是反复蹭热点。

示例：

```bash
cd /Users/whc/.codex/skills/MX_FinSearch
python -m scripts.get_data "股票名称 股票代码 最近12个月 公告 问询函 立案调查 诉讼 股权质押 重组 定增"
```

### 4. 补关键财务数据

对进入短名单的股票，用 `MX_FinData` 分批查询。每批最多 5 只股票、3 个指标。

建议指标批次：

- 批次 A：营业收入、归母净利润、扣非净利润。
- 批次 B：经营现金流净额、资产负债率、流动比率。
- 批次 C：ROE、毛利率、研发费用率。
- 批次 D：总市值、最新价、近 20 日成交额。
- 分红任务额外查：每股分红、股息率、分红率。

示例：

```bash
cd /Users/whc/.codex/skills/MX_FinData
python3 scripts/get_data.py --query "股票A 股票B 股票C 股票D 股票E 2021年至2025年 营业收入 归母净利润 扣非净利润"
```

### 5. 催化剂验证

用 `MX_FinSearch` 验证催化剂，而不是只看概念标签。

优先级从高到低：

1. 业绩拐点：订单、产品量产、产能释放、亏损收窄、毛利率改善。
2. 资本事件：重组、定增、国资入主、产业资本注资、股权激励。
3. 产业事件：政策、招标、价格变化、临床进展、产品认证。
4. 市场事件：财报窗口、解禁后压力释放、券商覆盖、机构调研。

如果催化剂只有“概念热门”，评级最多给“中”，并明确说明证据不足。

### 6. 评分与分层

用 0-5 分制，最后给总评，不要把分数伪装成确定性预测。

| 维度 | 评分要点 |
| --- | --- |
| 财务质量 | 现金流、负债率、ROE、扣非利润、商誉、应收和存货 |
| 成长空间 | 行业增速、国产替代、技术壁垒、客户质量 |
| 催化强度 | 事件明确性、时间窗口、公告证据、可验证里程碑 |
| 估值弹性 | 市值小、收入体量、同行估值差、业绩敏感度 |
| 交易质量 | 成交额、换手率、波动率、筹码结构 |
| 风险暴露 | 退市、监管、造假、质押、诉讼、融资压力 |

风险等级：

- **低**：基本面稳健，主要风险是估值和行业波动。
- **中**：存在波动或成长兑现压力，但无明显硬伤。
- **高**：未盈利、现金流弱、事件依赖强，需小仓位事件驱动。
- **极高**：监管/退市/诉讼/资金链风险明显；一般剔除，除非用户明确要求观察。

## 输出格式

### 小盘股 5/10 只筛选

用表格输出：

| 代码 | 名称 | 市值 | 行业/主题 | 核心催化 | 财务质量 | 为什么可能暴涨 | 最大风险 | 风险等级 | 定性 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |

表格后补充：

- **入选逻辑**：说明筛选条件和数据日期。
- **市场忽视点**：每只不超过 2 句。
- **剔除逻辑**：说明因 ST、监管、流动性、财务雷被排除的类型。
- **结论**：标注“投机 / 投资 / 事件驱动博弈”。

### 单票分析

按以下结构输出：

1. 商业模式：怎么赚钱，收入来源和客户结构。
2. 行业地位：竞争对手、份额、差异化。
3. 财务质量：收入、利润、现金流、负债、研发、应收/存货。
4. 估值对比：市值、PE/PB/PS，和同行对比。
5. 增长逻辑：未来 6 至 24 个月最关键变量。
6. 潜在风险：政策、行业、财务、治理、交易风险。
7. 多头逻辑：为什么可能涨。
8. 空头逻辑：为什么可能跌。
9. 当前更像：投机 / 投资 / 博弈。

## 合规边界

- 输出是研究和风险提示，不承诺收益，不给确定性买卖指令。
- 涉及当前股价、财务、公告、监管状态时，必须使用最新可获得数据源，并标明数据日期。
- 数据缺失时写“未获取到/需进一步核验”，不要补造。
