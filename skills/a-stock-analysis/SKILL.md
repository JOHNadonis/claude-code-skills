
- ❌ 关键叙事仅出自研报、未经公司官宣
- ❌ 机构持仓连续 2 期减持
- ❌ PE > 80 且营收增速 < 20%


**四因子评分标准**（总分 20）：
- Value（5分）：PE/PB 相对行业百分位
- Growth（5分）：营收+扣非净利双增速
- Momentum（5分）：60/120日板块内收益排名
- Quality（5分）：ROE + 负债率 + 现金流比

**评级阈值**：
- A（16-20分）→ 重仓
- B（12-15分）→ 轻仓
- C（8-11分）→ 观察
- D（4-7分）→ 不推荐
- E（否决）→ 剔除

**Step 3：机构共识度（L3，新增）**
```bash
cd ~/.Codex/skills/mx-finance-search && python3 scripts/get_data.py \
  --query "[股票名][代码] 北向资金 近5日 融资余额 公募持仓 险资持仓"
```

**共识度公式**：
```
共识度 = 险资×3 + 北向×2 + 公募×2 + 期指×1 - 融资×1
```

- ≥ +4 → 加仓
- ≤ -4 → 减仓
- -2~+2 → 持仓观望

**Step 4：产业链定位（L3，新增）**
```bash
cd ~/.Codex/skills/mx-finance-search && python3 scripts/get_data.py \

```

**必须标注**：
- 所属链条：Chain X（AI 算力 / 半导体 / ...）
- 所在环节：上游 / 中游 / 下游
- 链条地位：龙头 / 一线 / 二线 / 三线
- 领先指标状态：↑↓-

**Step 5：筹码结构（L4，新增）**
```bash
cd ~/.Codex/skills/mx-finance-search && python3 scripts/get_data.py \
  --query "[股票名][代码] 股东户数 解禁日期 解禁比例 控股股东增减持"
```

**筹码评分**：
- +3 股东户数连降 3 期以上
- +2 90天内无大额解禁（比例<5%）
- -5 大额解禁将至（比例>20%，60天内）
- -4 控股股东窗口期持续减持

**Step 6：衍生品信号（L4，新增）**
```bash
cd ~/.Codex/skills/mx-finance-search && python3 scripts/get_data.py \
  --query "股指期货 IM 年化基差率 50ETF期权 隐含波动率 科创板两融余额"
```

**三信号联合判读**：
- IM期货深贴水 + 期权IV低 + 融资动能衰减 → 减仓信号
- IM期货收敛 + 期权IV高 + 融资低位 → 左侧建仓信号

**Step 7：情绪象限识别（L4，新增）**

**当前象限判断**：
- 恐慌：VIX高 + 破净股多 + 融资低位
- 修复：低位放量 + 主力流入 + 北向加仓
- 亢奋：连板多 + 融资狂飙 + 成交额天量
- 崩塌：量价背离 + 龙头跌停 + 主力出逃

**对应策略**：
- 亢奋 → 拒绝追高，龙头逐步兑现
- 崩塌 → 减仓到 50%，只留防御
- 恐慌 → 左侧埋伏，只买低估值高股息
- 修复 → 从左侧切右侧，主力流入方向加仓

**Step 8：动态仓位计算（L5，新增）**

**半凯利公式**：
```
f = [(盈亏比×胜率 - 亏率) / 盈亏比] × 0.5
```

**当前参数**（待60天回测校准）：
- 胜率估计：55%（保守）
- 盈亏比：目标涨30% / 止损8% = 3.75
- 半凯利：≈ 21%
- 单票实际建议：15%（账户20000元 = 3000元/票）

**波动率调整**：
```
w = min(f, 目标波动率/标的波动率)
```

**单票上限**：25%（安全边界）

> **[不可省略]** Step 8 完成后，执行：
> - `mx-finance-search: "[股票名] 最新公告 消息 [今日日期]"`
> - 若有特定板块背景，追加：`mx-finance-search: "[板块] 板块动态 [今日日期]"`

## 模式二：盘中实盘辅助

### 盘中三阶段流水线

根据当前时间自动选择入口：

```
09:00-09:25（竞价前）→ 阶段A：盘前准备
```

## 做T内嵌决策（适宜度评估 + 时间窗口）

> 每次给做T建议时必须经过此决策树，不依赖 references 即可判断。

### Step 1：时间窗口过滤（先于一切）

```
当前时间                操作
09:30-09:45           ❌ 开盘剧烈波动期，不做T
09:45-11:30           ✅ 最优做T窗口（上午）
11:30-13:00           ❌ 午盘休市，无法做T
13:00-13:30           ⚠️  观察开盘情绪，谨慎操作
13:30-14:30           ✅ 次优做T窗口（下午）
14:30-15:00           ❌ 收盘前量化扫单时段，不做T
```

### Step 2：适宜度评分（必须逐项判断）

| 指标 | 高适宜（+） | 低适宜（-） | 权重 |
|------|-----------|-----------|------|
| 振幅 | 当日振幅 > 3% | 振幅 < 1.5%（横盘） | 高 |
| 量比 | 1.0-3.0（活跃） | < 0.8 或 > 3.0 | 高 |
| RSI | 65-80（有做T空间） | > 80 或 < 40 | 中 |
| BOLL | 上下轨间距宽 | 布林带收窄 | 中 |
| 资金 | 超大单双向活跃 | 超大单单向趋势强 | 中 |
| 消息 | 无重大公告 | 有未消化公告 | 高 |

综合：高适宜 ≥ 4项 → 适宜度「高」；2-3项 → 「中」；≤ 1项 → 「低」

### Step 3：做T量计算

```
做T量（卖出股数）= 总持仓 × 做T比例

做T比例参考：
  适宜度高 + 浮盈 > 5%    → 20-30%
  适宜度高 + 浮盈 2-5%    → 10-20%
  适宜度中                 → 5-10%
  适宜度低                 → 不做T
  持仓金额 < 2万           → 不做T（手续费覆盖不了收益）

⚠️ 做T后必须持有回补买入量 ≥ 卖出量（T+1限制，不能空仓）
```

### Step 4：两种策略选择

**策略A：技术位做T**（振幅>3%，RSI 65-80）
- 卖出：BOLL上轨 / 日内前高 / 涨幅 > +4%
- 回补：MA5 / 当日开盘价 / 日内低点 + 0.5%

**策略B：成本优化做T**（震荡盘整期）
- 触发：当前价 ≥ 成本 × 1.03
- 回补目标：≤ 成本 × 1.015

## 关键规则（实盘铁律）

> 格式：**触发条件 → 强制行动**。无强制行动 = 输出不完整。

| # | 触发条件 | 强制行动 |
|---|---------|---------|
| 1 | 量比 < 0.8 | 所有技术信号标"降级"，做T适宜度强制标"低"，不给激进买点 |
| 2 | 量比 < 0.5（地量） | 直接输出"等放量确认方向，今日不操作"，技术分析简化 |
| 3 | RSI > 70 | 不给追高建议，三档买点只给"稳健"和"强支撑"，注明"等回调" |
| 4 | RSI > 80 | 标"严重超买"，建议减仓而非做T |
| 5 | 竞价高开 + 量比 < 0.8 | 标"假高开/洗盘"，等开盘后实际价格确认再操作 |
| 6 | 距解禁 < 30天 | 止损收紧至 `成本×0.97`，持有评级降一星，必须在风险标记中显著标出 |
| 7 | 质押比 > 40% | 不加仓，持有评级降一星，标注"大股东质押风险" |
| 8 | 重大公告当日未消化 | 不给任何买卖操作建议，仅输出"等公告消化后再看" |
| 9 | 止损位 | 每次分析必须明确给出具体价格：`max(成本×0.95, MA20)` |
| 10 | 止盈位 | 每次分析必须明确给出具体价格：`min(BOLL上轨, 成本×1.15)` |
| 11 | 风险收益比 < 2:1 | 在买点下方标注"⚠️ 性价比偏低，建议等更好位置" |
| 12 | 超大单信号 vs 主力信号冲突 | 以超大单为准，主力可能含做市商，注明"信号分歧" |
| 13 | 连续3日主力净流出 | 不加仓，减仓或观望，注明"主力离场" |
| 14 | 单次超过5只股票 | 分批查询（每批≤5只），告知用户分批原因 |
| 15 | 消息面有重大利空 | 消息面优先于技术面，即便技术面强势也标"谨慎" |
| 16 | 仓位已满（用户有多只持仓） | 新买点建议前，先问"是否有计划腾仓位？优先卖哪只？" |

## 边界条件处理（实盘常见异常）

### API / 数据异常

| 异常情况 | 判断方式 | 处理方式 |
|---------|---------|---------|
| MX_FinData 返回空数据 | data 字段为空数组 | 先打印 raw JSON 确认 status 码；status=0 是成功，不要误判为失败 |
| 接口超时 / 网络错误 | 脚本报错 | 重试一次；仍失败则用 WebSearch 补充该股最新行情，注明"数据源降级" |
| 字段缺失（如无龙虎榜） | 字段值为 null | 跳过该字段，不要输出"数据获取失败"，直接写"未上榜" |
| 量比字段为 0 或 null | 非交易时段 | 注明"非交易时段，量比无效"，技术分析改用昨日数据 |

### 时段边界

| 时段 | 行为 |
|------|------|
| **09:00前** | 无法获取当日量比/竞价，买点分析基于前日收盘数据，明确注明 |
| **09:25-09:30**（集合竞价） | 只看竞价量比和价格偏离，不做T，等开盘确认 |
| **11:30-13:00**（午休） | 数据静态，可做买点分析，盘中操作等13:00后 |
| **15:00后**（盘后）| 自动进入阶段D复盘模式，不给当日操作建议，给明日预案 |
| **周末/节假日** | 明确提示"今日休市"，提供明日开盘前需关注的事项清单 |

### 股票特殊状态

| 状态 | 判断 | 处理 |
|------|------|------|
| **停牌** | 无成交量/无行情 | 输出"该股停牌，无法分析技术面"，只输出基本面风险（解禁/质押）|
| **ST/ST\*** | 股票名含ST | 额外标注"ST股风险：退市风险/流动性差"，不给买点 |
| **涨跌停** | 价格触及±10% | 涨停：不追，等次日竞价；跌停：不急止损，等开板确认 |
| **新股/次新股** | 上市<3个月 | 无法用历史均线/BOLL，注明"次新股，技术面参考价值低" |
| **超过5只股票** | 用户一次询问>5只 | 分批查询，每批≤5只，告知用户分批原因 |

## 模式三：主动预警

> **盯盘助手**：定时检测资金流向异动、公告风险、止损临近、大盘情绪，按分级告警推送给你。
> 告警等级：🔴 CRITICAL（立即处理） / 🟡 WARNING（需关注） / 🔵 INFO（参考信息）

### 触发方式

```
用户说："帮我盯盘" / "开启预警" / "有没有资金异动" / "止损了吗"
→ 执行一次性扫描：python3 scripts/monitor.py
→ 或只检测大盘：python3 scripts/monitor.py --market
→ 或只检测某只：python3 scripts/monitor.py 002463
→ 查看/修改阈值：python3 scripts/monitor.py --config
```

### 定时自动运行（PM2，交易日每小时触发）

```bash
# 安装后台守护（每个交易日 9/10/11/13/14 点运行一次）
pm2 start "cd ~/.Codex/skills/a-stock-analysis && python3 scripts/monitor.py" \
    --name a-stock-monitor --cron "0 9,10,11,13,14 * * 1-5" --no-autorestart
pm2 save
```

### 五类检测规则（每次扫描自动执行）

| 检测维度 | 触发条件 | 告警级别 |
|---------|---------|---------|
| **大盘** | 沪指跌 > 2% | 🔴 CRITICAL |
| **大盘** | 沪指跌 1-2% | 🟡 WARNING |
| **止损** | 当前价 ≤ max(成本×0.95, MA20) | 🔴 CRITICAL |
| **止损临近** | 当前价 ≤ 成本×0.97 | 🟡 WARNING |
| **资金流出** | 超大单净流出 > 3000万 | 🟡 WARNING |
| **资金流入** | 超大单净流入 > 3000万 | 🔵 INFO |
| **量比地量** | 量比 < 0.5 | 🟡 WARNING |
| **量比异动** | 量比 > 3.0 | 🔵 INFO |
| **利空公告** | 含：处罚/立案/亏损/停产/诉讼 | 🟡 WARNING |
| **利好公告** | 含：中标/回购/增持/分红/新品 | 🔵 INFO |

### 告警阈值自定义

修改 `data/alert_config.json` 覆盖默认阈值（不需要改的字段直接删除）：

```json
{
  "stop_loss_pct": 0.95,
  "capital_outflow_warn": -3000,
  "market_drop_critical": -2.0
}
```

### 扫描后 Codex 分析流程

1. 收到 monitor.py 输出的预警报告
2. 按告警等级从高到低逐条解读
3. 结合当前持仓成本和技术位给出操作建议
4. 若有 🔴 CRITICAL，必须优先给出止损/减仓的具体执行命令
