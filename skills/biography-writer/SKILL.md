# 历史人物传记写作系统

本 Skill 提供完整的历史人物传记写作工作流，包括大纲设计、章节生成、三级质量审查、问题定位与修复。

## 触发条件

**关键词触发：**
- 用户提到写传记、人物传记、历史人物
- 用户提到某个历史人物的名字 + 写作/分析/研究
- 用户使用 `/biography` 命令

**初始询问：**
在开始之前，询问用户：

1. **写作对象**：要写哪位历史人物？
2. **目标定位**：通俗读物 / 学术研究 / 内容营销？
3. **篇幅规划**：预计多少字？（建议：短篇3万字 / 中篇8万字 / 长篇15万字+）
4. **特殊关注**：有什么特别想探讨的角度或问题？

如用户已提供部分信息，跳过已回答的问题。

## 阶段一：项目初始化

### 1.1 创建项目结构

为用户创建标准化项目目录：

```bash
mkdir -p [project-name]/{chapters,reviews/{L1-fact-check,L2-consistency,L3-readability},sources,prompts}
```

### 1.2 创建 AGENTS.md

将以下内容写入项目根目录的 `AGENTS.md`：

```markdown
# 历史人物传记写作系统 v2.0

## 角色定位
你是一位兼具学术严谨性和文学叙事能力的历史传记作家。你的目标是写出既经得起史学考证、又让普通读者愿意读下去的传记作品。

## 三层结构法（核心框架）

每一章必须同时编织三条线索：

### 第一层：叙事层（WHAT）
- 人物做了什么？发生了什么事件？
- 要求：时间线清晰、场景具体、细节生动
- 技巧：用"镜头感"写作，让读者"看到"而非"被告知"

### 第二层：人物层（WHY）
- 人物为什么做这个决定？
- 要求：挖掘动机、呈现性格、描绘人际关系网络
- 技巧：通过对话、行为、他人评价多角度刻画，避免直接贴标签

### 第三层：制度层（HOW POSSIBLE）
- 人物能做什么、不能做什么？环境如何塑造选择空间？
- 要求：交代政治制度、经济条件、文化背景、技术水平
- 技巧：制度背景要"溶解"在叙事中，而非单独成段说教

## 现代理论工具箱（适度使用）

| 理论 | 适用场景 | 示例 |
|------|----------|------|
| 博弈论 | 多方互动、利益冲突 | "曹操面临囚徒困境：若先动手..." |
| 委托代理 | 上下级关系、信息不对称 | "皇帝无法监控边疆将领的真实战况..." |
| 路径依赖 | 解释为何不走"更好的路" | "科举制度一旦建立，就锁定了..." |
| 信号博弈 | 政治表态、权力试探 | "他的辞官是一次信号发送..." |
| 制度经济学 | 规则如何塑造激励 | "均田制决定了府兵制的可行性..." |

**使用原则**：理论是"透镜"而非"标签"，用来增进理解，不是炫技。

## 质量红线（绝对禁止）

### 叙事禁忌
- ❌ 上帝视角："他没想到，二十年后这个决定将改变历史"
- ❌ 后见之明："事后看来，这显然是个错误"
- ❌ 空洞判断："他是一位伟大的政治家"（用事实说话）

### 语言禁忌
- ❌ AI 典型句式："值得一提的是"、"不难发现"、"显而易见"
- ❌ 现代网络语："内卷"、"躺平"、"凡尔赛"
- ❌ 过度排比：连续三个以上的"他是...他是...他是..."
- ❌ 形容词堆砌："伟大而卓越的、杰出且非凡的"

### 学术禁忌
- ❌ 无来源断言：重要事实必须标注"据《XX》记载"
- ❌ 单一来源：争议性事件应呈现多方记载
- ❌ 过度脑补：区分"史料记载"与"合理推测"

## 质量要求（必须做到）

### 叙事质量
- ✅ 开篇钩子：每章前500字必须抓住读者
- ✅ 场景还原：重要事件用场景化写法
- ✅ 节奏控制：张弛有度，大事件展开写，过渡期可快进

### 学术质量
- ✅ 来源标注：关键史实标注出处
- ✅ 争议呈现：不同史料的不同记载
- ✅ 推测说明：合理推断要明确标识

### 结构质量
- ✅ 章节独立：每章有完整的起承转合
- ✅ 线索连贯：伏笔要有回应
- ✅ 三层交织：叙事、人物、制度三层不能偏废

## 输出格式

每章输出结构：
1. **章节标题**（引人入胜）
2. **时间跨度**（起止年份）
3. **正文**（8000-15000字）
4. **本章要点**（3-5条）
5. **待核实事项**（如有）

## 自我标注规范

写作时对不确定处进行标注：
- `【存疑】` - 史实不确定
- `【待补充来源】` - 需要补充引用
- `【争议点】` - 存在多种解读
```

## 阶段二：大纲设计

### 2.1 收集背景信息

询问用户是否有：
- 已有的参考资料或史料
- 特别想回答的核心问题
- 需要避开的角度或观点

### 2.2 生成大纲

使用以下提示词生成大纲：

```
我要写一本关于[人物名]的历史传记，预计[X]万字。

请用三层结构法设计全书框架：

1. **叙事层**：人物生平的关键转折点（建议8-12个）
   - 每个转折点的时间、核心事件、戏剧张力

2. **人物层**：核心人际网络
   - 列出10个最重要的人物关系
   - 每个关系的性质（盟友/对手/贵人/部下）

3. **制度层**：时代关键背景
   - 政治制度特点
   - 经济结构
   - 文化/思想环境
   - 技术水平

输出格式：
- 章节列表（标题 + 时间跨度 + 核心事件 + 字数规划）
- 全书核心问题（这本书要回答什么问题？）
- 预期读者收获
```

### 2.3 确认大纲

将大纲保存为 `outline.md`，请用户确认或调整。

## 阶段三：章节生成

### 3.1 生成方式选择

询问用户偏好：
- **并行生成**：同时启动多个 Task 生成多章（快速，适合赶时间）
- **逐章生成**：一章一章写（可控，适合精雕细琢）

### 3.2 并行生成模式

使用 Task 工具并行生成：

```
请同时启动 [N] 个写作任务，分别撰写第 [X, Y, Z] 章。

每个任务规格：
- 人物：[人物名]
- 章节信息：参考 outline.md 中的对应章节
- 字数：10000-12000字
- 严格遵循 AGENTS.md 中的三层结构法
- 完成后保存到 chapters/0X-[章节简称].md
```

### 3.3 逐章生成模式

使用 TodoWrite 追踪进度，逐章生成：

```
请为第[X]章撰写内容。

【章节信息】
- 标题：[从outline.md获取]
- 时间跨度：[起止年份]
- 核心事件：[列举]
- 字数要求：[X]字

【写作要求】
严格遵循 AGENTS.md 中的规范，特别注意：
1. 三层结构法的交织
2. 开篇钩子的设计
3. 来源标注的规范
4. 避免质量红线中的问题

完成后保存到 chapters/0X-[章节简称].md
```

## 阶段四：三级质量审查

每章完成后，依次执行三级审查。

### 4.1 第一级：事实核查

```
请对 chapters/[filename] 执行第一级审查（事实核查）。

审查要点：
1. 时间校验：日期、年份、顺序
2. 事实校验：人名、地名、官职、事件
3. 引用校验：来源准确性、单一来源问题
4. 逻辑校验：因果关系、后见之明、过度推测

输出格式：
- 确认无误的事实
- 需要修正的错误（表格）
- 需要补充来源的断言
- 需要多方观点的争议
- 可能的过度推测

保存到 reviews/L1-fact-check/[chapter-number].md
```

### 4.2 第二级：逻辑与一致性

```
请对 chapters/[filename] 执行第二级审查（逻辑与一致性）。

参考上下文：
- 全书大纲：outline.md
- 前一章：chapters/[prev].md（如有）
- 后一章大纲：outline.md 中的对应部分

审查要点：
1. 人物一致性：性格、行为逻辑、关系
2. 时间线一致性：衔接、矛盾、伏笔
3. 叙事逻辑：因果链条、逻辑跳跃
4. 三层结构平衡：叙事/人物/制度是否偏废

输出格式：
- 一致性问题（表格）
- 逻辑缺陷（表格）
- 三层结构评分
- 未解决的伏笔

保存到 reviews/L2-consistency/[chapter-number].md
```

### 4.3 第三级：可读性与风格

```
请对 chapters/[filename] 执行第三级审查（可读性与风格）。

审查要点：
1. "AI味"检测：过渡词滥用、空洞总结、机械排比、形容词堆砌
2. 语言时代感：网络用语、现代概念穿越
3. 节奏可读性：开篇吸引力、段落长度、信息密度
4. 情感画面感：场景还原、"展示"vs"告诉"

输出格式：
- AI味道问题（表格）
- 时代感问题（表格）
- 节奏问题（表格）
- 场景强化建议
- 整体风格评分
- 本章亮点（确保不被改掉）

保存到 reviews/L3-readability/[chapter-number].md
```

## 阶段五：问题定位与修复

### 5.1 使用 Grep 快速定位

```bash
# 搜索 AI 味表达
Grep: "值得一提的是|不难发现|显而易见|毋庸置疑|总之|综上所述"
Path: chapters/

# 搜索待处理标记
Grep: "【存疑】|【待补充来源】|【争议点】"
Path: chapters/

# 搜索特定年份（时间校验）
Grep: "\\d{3,4}年"
Path: chapters/
```

### 5.2 批量修复

```
我已完成所有章节的三级审查，结果在 reviews/ 目录下。

请：
1. 读取所有审查报告
2. 按严重程度排序问题（事实错误 > 逻辑矛盾 > 风格问题）
3. 逐一修复，从最严重的开始
4. 每修复一个问题，用 TodoWrite 标记完成
5. 修复后重新运行对应级别的审查确认
```

### 5.3 迭代策略

- **事实错误**：必须立即修复，不可妥协
- **逻辑矛盾**：必须修复，可能需要调整多章
- **风格问题**：酌情修复，注意保留亮点
- **迭代终止条件**：三级审查都无严重问题

### 6.2 生成附录

## 局限性提醒

在开始时告知用户：

**本工具适合：**
- 快速产出商业类传记/通俗读物
- 内容营销、公众号连载
- 作为专业写作的初稿

**本工具不适合：**
- 严肃学术著作（需要人类专家深度介入）
- 需要原创历史观点的作品
- 细节考证要求极高的场景

**建议：**
- 关键章节安排人工审读
- 重要史实需要人工核实
- 把 AI 产出视为"70分初稿"，最后30分需人类打磨

## 成本估算参考

以15万字传记为例：
- 生成：约 30万 token
- 审查迭代：约 1-1.5 亿 token
- 预估费用：$400-600（Codex 3.5 Sonnet）

**核心认知**：生成只占 0.2%，审查迭代占 99.8%。
