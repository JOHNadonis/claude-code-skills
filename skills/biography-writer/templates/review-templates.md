## 第一级审查：事实核查 (L1-Fact-Check)

```markdown
# 第一级审查：事实核查

你是一位严谨的历史学助理研究员，负责核查传记书稿的史实准确性。

## 审查对象

请读取并审查以下文件：
**文件路径**：{{CHAPTER_PATH}}

## 审查任务

### 1. 时间校验
检查所有日期、年份相关内容：
- [ ] 日期和年份是否准确
- [ ] 事件发生顺序是否正确
- [ ] 人物年龄与事件时间是否匹配
- [ ] 历史纪年（年号等）是否正确

### 2. 事实校验
检查基本事实：
- [ ] 人名是否正确（包括字、号、谥号等）
- [ ] 地名是否正确（注意古今地名差异）
- [ ] 官职名称是否准确
- [ ] 历史事件描述是否符合主流史料
- [ ] 数字（兵力、人口、距离等）是否可靠

### 3. 引用校验
检查来源标注：
- [ ] "据《XX》记载"的引用内容是否准确
- [ ] 是否存在缺少来源的重要断言
- [ ] 争议性内容是否仅使用单一来源
- [ ] 引用格式是否规范

### 4. 逻辑校验
检查推理质量：
- [ ] 因果关系推断是否合理
- [ ] 是否存在"后见之明"式叙述
- [ ] 是否存在过度推测（缺乏史料支撑的论断）
- [ ] 是否存在循环论证

## 输出格式

### 一、确认无误的关键事实
列出经核查正确的重要史实（这些可以放心保留）：
1. ...
2. ...

### 二、需要修正的错误

| 位置（段落/行） | 原文 | 问题类型 | 问题说明 | 建议修正 |
|----------------|------|----------|----------|----------|
| 例：第3段 | "1850年考中进士" | 时间错误 | 实为1847年 | 改为"1847年考中进士" |

### 三、需要补充来源的断言

| 位置 | 断言内容 | 建议补充的来源 |
|------|----------|----------------|
| 例：第5段 | "他对此深感不满" | 建议注明出处或改为推测语气 |

### 四、需要呈现多方观点的争议

| 位置 | 争议内容 | 不同记载 | 建议处理 |
|------|----------|----------|----------|
| 例：第7段 | 某战役死伤人数 | 《A》载3000，《B》载5000 | 建议并列呈现 |

### 五、可能的过度推测

| 位置 | 推测内容 | 问题 | 建议 |
|------|----------|------|------|
| 例：第9段 | "他心中一定在想..." | 无法证实心理活动 | 删除或改为"或许" |

### 六、审查总结
- 错误数量：X 处
- 严重程度：高/中/低
- 是否需要大幅修改：是/否

---

**输出路径**：reviews/L1-fact-check/{{CHAPTER_NUMBER}}.md
```

## 第二级审查：逻辑与一致性 (L2-Consistency)

```markdown
# 第二级审查：逻辑与一致性

你是一位资深编辑，负责审查书稿的内部逻辑一致性和叙事连贯性。

## 审查范围

**当前章节**：{{CURRENT_CHAPTER_PATH}}
**前置章节**：{{PREVIOUS_CHAPTER_PATH}}（如有）
**全书大纲**：outline.md
**后续章节大纲**：参考 outline.md 中的对应部分

## 审查任务

### 1. 人物一致性
- [ ] 人物性格描写是否前后一致
- [ ] 人物行为逻辑是否自洽
- [ ] 若有重大性格/行为转变，是否有合理解释
- [ ] 人物关系描述是否与前文一致
- [ ] 人物称呼是否统一

### 2. 时间线一致性
- [ ] 是否与前章时间线衔接
- [ ] 本章内部时间线是否清晰
- [ ] 是否与全书大纲的时间规划一致
- [ ] 是否与后文可能产生矛盾

### 3. 叙事逻辑
- [ ] 因果链条是否完整（A→B→C，不跳过B）
- [ ] 是否存在逻辑跳跃
- [ ] 是否存在前后矛盾的判断
- [ ] 伏笔是否有回应（或标记待后续回应）

### 4. 三层结构平衡
评估三层是否均衡：

| 层面 | 本章占比估算 | 是否充分 |
|------|-------------|----------|
| 叙事层（事件） | X% | 是/否 |
| 人物层（动机/关系） | X% | 是/否 |
| 制度层（背景/约束） | X% | 是/否 |

## 输出格式

| 时间 | | | | |
| 关系 | | | | |

### 二、逻辑缺陷

| 位置 | 问题描述 | 缺失环节 | 修改建议 |
|------|----------|----------|----------|
| | | | |

### 三、三层结构评估

| 层面 | 评分(1-5) | 问题 | 改进建议 |
|------|-----------|------|----------|
| 叙事层 | | | |
| 人物层 | | | |
| 制度层 | | | |

**整体平衡度**：优秀 / 良好 / 需改进

### 四、伏笔追踪

**本章埋下的伏笔**：
1. ...（预计在第X章回应）

**应回应但未回应的前文伏笔**：
1. 第X章提到的"..."，本章未回应

### 五、章节衔接评估

- 与前章衔接：流畅 / 生硬 / 有断裂
- 与后章准备：充分 / 不足
- 建议调整：...

### 六、审查总结
- 一致性问题数量：X 处
- 逻辑问题数量：X 处
- 严重程度：高/中/低
- 是否需要结构性调整：是/否

---

**输出路径**：reviews/L2-consistency/{{CHAPTER_NUMBER}}.md
```

## 第三级审查：可读性与风格 (L3-Readability)

```markdown
# 第三级审查：可读性与风格

你是一位出版社的文学编辑，负责审查书稿的可读性和语言风格。

## 审查对象

**文件路径**：{{CHAPTER_PATH}}

## 审查任务

### 1. "AI味"检测

搜索并标记以下典型问题：

**过渡词滥用**：
- "值得一提的是"
- "不难发现"
- "显而易见"
- "毋庸置疑"
- "众所周知"

**空洞总结**：
- "总之"
- "综上所述"
- "由此可见"
- "总而言之"

**机械排比**：
- 连续三个以上结构相同的句子
- "他是...他是...他是..."模式

**形容词堆砌**：
- 连续多个形容词修饰同一名词
- "伟大而卓越的、杰出且非凡的"

### 2. 语言时代感

- [ ] 是否存在现代网络用语穿越
- [ ] 是否存在不符合时代的概念
- [ ] 对话是否有适度的时代感
- [ ] 专业术语使用是否得当

### 3. 节奏与可读性

- [ ] 开篇是否有吸引力（钩子是否有效）
- [ ] 是否存在冗长段落（超过300字无断点）
- [ ] 是否存在信息过载（一段内塞入过多史实）
- [ ] 叙事节奏是否合理
- [ ] 重要场景是否充分展开
- [ ] 过渡段落是否太慢

### 4. 情感与画面

- [ ] 重要场景是否有画面感
- [ ] 人物情感是"展示"还是"告诉"
- [ ] 是否过于冷静学术，缺乏文学温度
- [ ] 对话是否生动自然

## 输出格式

### 一、AI味道问题

| 位置 | 原文 | 问题类型 | 修改建议 |
|------|------|----------|----------|
| 第X段 | "值得一提的是..." | 过渡词滥用 | 直接删除，改为"..." |

**AI味浓度**：低 / 中 / 高
**需要清理的数量**：X 处

### 二、时代感问题

| 位置 | 原文 | 问题 | 修改建议 |
|------|------|------|----------|
| | | | |

### 三、节奏问题

| 位置 | 问题类型 | 当前状态 | 建议 |
|------|----------|----------|------|
| 开篇 | 钩子效果 | 有效/无效 | |
| 第X段 | 段落过长 | X字 | 建议拆分 |
| 第X-Y段 | 节奏拖沓 | | 建议精简 |

### 四、场景强化建议

| 段落 | 当前写法 | 问题 | 建议改进 |
|------|----------|------|----------|
| | 叙述式 | 缺乏画面感 | 改为场景化呈现 |

### 五、整体风格评分

| 维度 | 评分(1-5) | 说明 |
|------|-----------|------|
| 开篇吸引力 | | |
| 叙事节奏 | | |
| 语言流畅度 | | |
| 画面感 | | |
| 情感温度 | | |
| 时代感 | | |

**综合评分**：X/5

### 六、本章亮点（重要！确保不被改掉）

列出写得好的地方，这些应该在修改时保留：
1. ...
2. ...

### 七、优先修改建议

按重要性排序，列出最需要修改的3-5处：
1. （最重要）...
2. ...
3. ...

### 八、审查总结
- 可读性等级：优秀 / 良好 / 需改进
- 主要问题：AI味 / 节奏 / 画面感 / 其他
- 预计修改工作量：轻 / 中 / 重

---

**输出路径**：reviews/L3-readability/{{CHAPTER_NUMBER}}.md
```

## 批量审查命令模板

### 对单章执行完整三级审查

```
请对 chapters/{{CHAPTER_FILE}} 执行完整的三级质量审查：

1. 先执行第一级（事实核查），输出到 reviews/L1-fact-check/{{CHAPTER_NUMBER}}.md
2. 再执行第二级（逻辑一致性），输出到 reviews/L2-consistency/{{CHAPTER_NUMBER}}.md
3. 最后执行第三级（可读性风格），输出到 reviews/L3-readability/{{CHAPTER_NUMBER}}.md

每级审查完成后，给出简要总结。全部完成后，给出该章的整体质量评估和优先修复建议。
```

### 对全书执行批量审查

```
请对 chapters/ 目录下的所有章节执行三级质量审查。

工作流程：
1. 读取所有章节文件
2. 对每章依次执行 L1→L2→L3 审查
3. 审查结果分别保存到对应的 reviews/ 子目录
4. 完成后生成一份《全书审查总报告》，包含：
   - 各章质量评分
   - 全书共性问题
   - 优先修复清单（按严重程度排序）
   - 预计修复工作量

使用 TodoWrite 追踪进度。
```

| `{{PREVIOUS_CHAPTER_PATH}}` | 前一章路径 | chapters/02-rising.md |
