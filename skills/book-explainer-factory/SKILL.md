# 读书博主深度讲书内容生产线（70% 展开版）

> 核心原则：**不是"留悬念让人买书"，是"把核心逻辑链讲明白，让书来补细节和原文体验"。**
> 你的工作是当"领路人"——带观众走完主干道，让他们自己去探索小巷。观众看完的状态应该是：「逻辑我跟上了，但要真正吃透，得自己读一遍。」

## 与其他 skill 的关系

| Skill | 解决什么 | 关系 |
|-------|---------|------|
| 本 skill | 深度讲书（70% 展开，重逻辑链完整性） | 核心 |
| `/book-seller-factory` | 带货卖书（30% 展开，重知识缺口） | 对照版 |
| `/dontbesilent-writer` | 怎么说（文风润色） | 可串联 |
| `/non-consensus-factory` | 纯观点内容（不带货） | 不同目标 |

---

## 输入

- **书名**（必填）：书名、或书的内容摘要/截图/PDF
- **输出格式**：短视频口播稿（默认）/ 图文帖子
- **时长版本**：3 分钟（默认）/ 5 分钟 / 10 分钟
- **结构类型**：困惑-书解（默认）/ 场景代入 / 翻书实录 / 自动选择
- **数量**：默认 5 条选题

---

## 三种时长版本

> 不同时长不是"同一个稿子拉长缩短"，而是**展开的层数不同**。
> 70% 版的每个时长都比 30% 版多展开 1-2 层，留给书的是细节论证和原文质感。

| 时长 | 字数 | 展开层数 | 70%法则怎么用 | 适合平台 |
|------|------|---------|-------------|---------|
| **3 分钟** | 800-1000 字 | 破 + 立第一层 | 破刻板印象 + 展开框架第一层的完整定义 | 抖音/小红书/视频号 |
| **5 分钟** | 1500-2000 字 | 破 + 立两层 | 破刻板印象 + 展开两层推导链 | 视频号/B站/小红书长视频 |
| **10 分钟** | 3000-4000 字 | 破 + 立三层 + 留尾 | 破 + 展开三层推导 + 最后的细节论证留给书 | B站/播客/YouTube |

### 3 分钟版的结构节奏

```
0:00-0:15  Hook（反常识断言）
0:15-0:45  展示刻板印象 + 一个反例击碎
0:45-1:00  引出书
1:00-2:00  展开框架第一层（完整定义入口概念，带例子说明）
2:00-2:20  点明"这只是书的骨架，血肉在原文里"
2:20-2:30  收尾短句
```

关键：比 30% 版多了完整的第一层展开。观众不仅知道"旧的错了"，还清楚看到了"新的第一层长什么样"。留给书的是：这一层背后的细节论证和后续层次。

### 5 分钟版的结构节奏

```
0:00-0:20  Hook
0:20-1:00  展示刻板印象 + 反例
1:00-1:20  引出书
1:20-2:30  展开框架第一层（完整定义 + 例子）
2:30-3:40  展开框架第二层（深入推导，展示逻辑链如何从第一层走到第二层）
3:40-4:20  点明留给书的部分："知道每一步说了什么 ≠ 理解为什么成立"
4:20-4:40  收尾
```

关键：观众跟着走完了两层推导，看到了完整的逻辑骨架。留给书的是：每一步的细节论证、原文的表达质感、以及更深层次的展开。

### 10 分钟版的结构节奏

```
0:00-0:30   Hook
0:30-1:30   展示刻板印象 + 多个反例/数据
1:30-2:00   引出书
2:00-4:00   展开框架第一层（完整定义 + 多个例子 + 与常识的对比）
4:00-6:00   展开框架第二层（推导过程 + 关键转折点）
6:00-8:00   展开框架第三层（核心结论的推导 + 为什么这个结论重要）
8:00-9:00   点明留给书的部分 + 个人感受
9:00-9:30   为什么这本书值得自己读一遍
9:30-10:00  收尾
```

关键：观众跟着走完了三层推导，对书的核心论证有了完整理解。留给书的是：论证的细节质感、原文的思维密度、以及读者自己的"啊哈时刻"——这些只有亲自阅读才能获得。

## 三种讲书结构

### 结构 A：困惑-书解（默认）

**逻辑**：我曾经想不通一个问题 → 展示我的错误理解 → "直到我看了这本书" → 展开书里框架的核心推导链（70%），留细节论证给书

**模板**：

```
1. 【自我暴露】我一直以为 [常见认知]
2. 【碰壁】直到 [某个事实/经历] 让我发现这个认知是错的
3. 【转折点】后来看到《书名》里的一个说法，一下就通了
4. 【展开推导链】书里的逻辑是这样的：[第一层] → [第二层] → [核心结论]
5. 【留深度钩子】但知道这条链"说了什么"和真正理解"为什么成立"是两回事
6. 【收尾】这本书的原文推导比我讲的精密得多，值得自己走一遍
```

**关键**：
- 第 4 步展开完整的推导链，让观众看到逻辑骨架
- 第 5 步的钩子不是"还有更多没讲"，而是"理解深度还差一层"
- 你的角色是"领路人"——带观众走完主干道

---

### 结构 B：场景代入

**逻辑**：从观众日常困惑切入 → 证明常见解释是错的 → "这本书里有一整套解释体系" → 展开体系的核心推导链

**模板**：

```
1. 【日常困惑】你有没有想过，为什么 [观众身边的现象]？
2. 【常见解释】大部分人会说 [刻板印象]
3. 【击碎】但你只要想一下 [一个反例]，就发现这个解释站不住
4. 【引出书】《书名》里给了一个完全不同的解释框架
5. 【展开推导链】它的逻辑链是：[第一层定义] → [第二层推导] → [核心结论]
6. 【留深度钩子】我讲的是骨架，书里每一步都有严密的论证过程——那个过程才是真正烧脑的
7. 【收尾】[一句短句收]
```

**关键**：
- 第 1 步必须是观众真正困惑过的问题
- 第 5 步展开完整推导链，不在入口处停下
- 第 6 步的钩子是"骨架 vs 血肉"——你给了骨架，血肉在书里

---

### 结构 C：翻书实录

**逻辑**：直接拿着书 → "翻到这一页我愣住了" → 读一段原文 → 展开这段话背后的完整逻辑 → "但原文的推导密度是我转述不了的"

**模板**：

```
1. 【拿起书】这本《书名》我翻到第 X 页的时候停下来了
2. 【读原文】它说了一句话：「原文引用」
3. 【我的反应】我第一反应是 [震惊/不同意/突然理解了什么]
4. 【为什么这句话重要】因为 [连接到观众认知]
5. 【展开推导链】这句话背后的逻辑是：[第一层] → [第二层] → [结论]
6. 【留深度钩子】我用大白话讲了逻辑链，但原文每一步的论证密度是转述不了的
7. 【收尾】这种书，看别人讲和自己读，完全是两种体验
```

**关键**：
- 原文引用选"一句话就能制造认知冲击"的段落
- 第 5 步不在引用处停下，而是展开这句话背后的完整逻辑
- 你的角色是"已经读透的人在做导览"，不是"正在被改变的读者"

## 执行流程

### Step 1：拆书——提取"讲解级"论点

不是提取"能让人想买书的论点"，而是提取"能展开完整推导链、展开后仍有细节留给书的论点"。

**筛选标准**（全部满足才算讲解级）：
- ✅ 有完整的多层推导链（不是一个孤立观点）
- ✅ 推导链可以用大白话讲清楚（不需要专业术语才能理解）
- ✅ 展开 70% 后，剩下的 30%（细节论证、原文质感）仍有独立价值
- ✅ 和观众的日常生活/困惑有关联

**不合格的论点**：
- ❌ 没有推导过程，只有结论（展开不了）
- ❌ 推导链太短，一展开就 100% 了（没有留给书的空间）
- ❌ 必须依赖专业术语才能讲清楚（观众跟不上）

### Step 2：为每个论点选择最佳结构

| 论点特征 | 推荐结构 |
|----------|---------|
| 你自己曾经也搞错过 | A：困惑-书解 |
| 跟热点/日常话题直接相关 | B：场景代入 |
| 书里有一句特别炸裂的原话 | C：翻书实录 |

### Step 3：生成内容骨架

每条选题输出：

```markdown
### 选题 N：[标题]
- 结构：A / B / C
- 刻板印象：[要破的认知]
- 书的论点：[书怎么说的]
- 展开 70%：[你在内容里讲到哪一层，讲什么]
- 留给书的 30%：[细节论证、原文质感、更深层展开]
- 深度钩子：[用什么话制造"得自己读"的感觉]
```

### Step 4：展开为完整内容

**70% 法则**（核心铁律）：
- 讲清楚「旧认知为什么错」— 讲透
- 讲清楚「新框架的核心推导链」— 展开到倒数第二层，每层给定义和例子
- **留给书的 30%**：最后一层的细节论证、原文的表达质感、读者自己的顿悟时刻
- 你的内容结束时，观众的状态应该是：「逻辑链我跟上了，但要真正吃透每一步为什么成立，得自己读一遍」

## 质量自检

每条内容过以下检查：

- [ ] 核心推导链是否完整？观众能否跟着走完主干逻辑？
- [ ] 你的角色是"领路人"还是"学术讲师"？（必须是前者，用大白话）
- [ ] 留给书的 30% 是否有独立价值？（不是边角料，是细节论证和原文体验）
- [ ] 深度钩子是否到位？（"知道说了什么 ≠ 理解为什么成立"）
- [ ] 刻板印象是不是大众真的相信的？
- [ ] 引书是否自然？
- [ ] 70% 法则：是否展开了推导链但没有替代原文的阅读体验？

---

## 批量模式

1. 先输出所有选题的骨架（标题 + 结构 + 70%/30% 划分）
2. 用户挑选要展开的
3. 逐条展开
