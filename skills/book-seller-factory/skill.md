# 读书博主带货内容生产线

> 核心原则：**不是"讲完观点顺便推书"，是"制造知识缺口然后让书来满足"。**
> 内容讲得越透，转化越低。你的工作不是当老师，是当"书的销售员"——让观众觉得"我必须自己看这本书"。

## 与其他 skill 的关系

| Skill | 解决什么 | 关系 |
|-------|---------|------|
| 本 skill | 说什么 + 怎么卖书（选题 + 带货结构） | 核心 |
| `/dontbesilent-writer` | 怎么说（文风润色） | 可串联 |
| `/non-consensus-factory` | 纯观点内容（不带货） | 不同目标 |

---

## 输入

- **书名**（必填）：书名、或书的内容摘要/截图/PDF
- **输出格式**：短视频口播稿（默认）/ 图文帖子
- **时长版本**：3 分钟（默认）/ 5 分钟 / 10 分钟
- **结构类型**：困惑-书解（默认）/ 场景代入 / 翻书实录 / 自动选择
- **数量**：默认 5 条选题

---

## 三种时长版本

> 不同时长不是"同一个稿子拉长缩短"，而是**展开的层数不同**。
> 层数越多，知识缺口越深，但对创作者的讲述能力要求也越高。

| 时长 | 字数 | 展开层数 | 30%法则怎么用 | 适合平台 |
|------|------|---------|-------------|---------|
| **3 分钟** | 800-1000 字 | 只破，不立 | 只破一个刻板印象 + 给框架名字就停 | 抖音/小红书/视频号 |
| **5 分钟** | 1500-2000 字 | 破 + 立第一层 | 破刻板印象 + 展开框架入口/第一层 | 视频号/B站/小红书长视频 |
| **10 分钟** | 3000-4000 字 | 破 + 立两层 + 留尾 | 破 + 展开两层推导 + 最后一层明确留给书 | B站/播客/YouTube |

### 3 分钟版的结构节奏

```
0:00-0:15  Hook（反常识断言）
0:15-1:00  展示刻板印象 + 一个反例击碎
1:00-1:45  引出书 + 只给框架的名字/一句话概括
1:45-2:15  制造未完成感："但这只是书里的第一页"
2:15-2:30  收尾短句
```

关键：节奏快，信息密度高，不废话。一个刻板印象、一个反例、一个框架名字，结束。观众的状态是"等等，什么意思？我得去看看"。

### 5 分钟版的结构节奏

```
0:00-0:20  Hook
0:20-1:30  展示刻板印象 + 反例（可以多一个反例或数据）
1:30-2:00  引出书
2:00-3:30  展开框架的第一层（解释入口概念，但不定义子概念）
3:30-4:15  制造未完成感："后面还拆了 N 层"
4:15-4:40  收尾
```

关键：比 3 分钟版多了"第一层展开"。观众不仅知道"旧的错了"，还模糊看到了"新的长什么样"，但看不清细节。

### 10 分钟版的结构节奏

```
0:00-0:30   Hook
0:30-2:00   展示刻板印象 + 多个反例/数据
2:00-2:30   引出书
2:30-5:00   展开框架第一层（完整定义入口概念）
5:00-7:30   展开框架第二层（深入一步，展示推导过程）
7:30-8:30   明确留钩子："但最关键的第三层/结论我讲不了"
8:30-9:30   个人感受/为什么这本书值得
9:30-10:00  收尾
```

关键：观众跟着你走了两层推导，已经"上瘾了"，但在最关键的节点被告知"后面的得自己去看"。这是转化最强的版本，但要求创作者真的能讲清楚两层推导。

## 三种带货结构

### 结构 A：困惑-书解（默认，转化最高）

**逻辑**：我曾经想不通一个问题 → 展示我的错误理解 → "直到我看了这本书" → 只给书里框架的 30%，留 70% 给书

**模板**：

```
1. 【自我暴露】我一直以为 [常见认知]
2. 【碰壁】直到 [某个事实/经历] 让我发现这个认知是错的
3. 【转折点】后来看到《书名》里的一个说法，一下就通了
4. 【只给 30%】书里说 [核心框架的入口概念]
5. 【留钩子】但这只是第一层，书里后面还拆了 [N] 层，我今天讲不完
6. 【收尾】想真正搞懂 [这个问题]，这本书是我目前看过讲得最透的
```

**关键**：
- 第 4 步只讲框架的"入口"，不展开定义
- 第 5 步明确告诉观众"还有更多"，制造未完成感
- 你的角色是"受益者"，不是"老师"

---

### 结构 B：场景代入

**逻辑**：从观众日常困惑切入 → 证明常见解释是错的 → "这本书里有一整套解释体系" → 只展示体系的入口

**模板**：

```
1. 【日常困惑】你有没有想过，为什么 [观众身边的现象]？
2. 【常见解释】大部分人会说 [刻板印象]
3. 【击碎】但你只要想一下 [一个反例]，就发现这个解释站不住
4. 【引出书】《书名》里给了一个完全不同的解释框架
5. 【只给入口】它说 [框架的第一层]，光这一层就够颠覆的了
6. 【留钩子】但真正厉害的是后面几层推导，这里讲不了——自己去看
7. 【收尾】[一句短句收]
```

**关键**：
- 第 1 步必须是观众真正困惑过的问题，不是学术问题
- 第 3 步的反例要"一击必杀"，一个就够
- 第 5 步只给第一层，明确说"后面还有"

---

### 结构 C：翻书实录

**逻辑**：直接拿着书 → "翻到这一页我愣住了" → 读一段原文 → 展开自己的反应 → "但后面还有三层没讲"

**模板**：

```
1. 【拿起书】这本《书名》我翻到第 X 页的时候停下来了
2. 【读原文】它说了一句话：「原文引用」
3. 【我的反应】我第一反应是 [震惊/不同意/突然理解了什么]
4. 【为什么这句话重要】因为 [连接到观众认知]
5. 【只展开一层】简单说就是 [框架入口]
6. 【留钩子】但这只是这一章的开头，后面的推导才是真正炸裂的部分
7. 【收尾】这种书不是看一遍就够的
```

**关键**：
- 最适合短视频的视觉形式（手持翻书）
- 原文引用要选"一句话就能制造认知冲击"的段落
- 你的角色是"正在被这本书改变的读者"，不是"已经读完的专家"

## 执行流程

### Step 1：拆书——提取"卖点级"论点

不是提取"核心观点"，而是提取"能让人想买书的论点"。

**筛选标准**（全部满足才算卖点级）：
- ✅ 打破了一个大众真正相信的刻板印象
- ✅ 书里有完整的推导过程（不是一句金句能讲完的）
- ✅ 和观众的日常生活/困惑有关联
- ✅ 30% 就足够吸引人，但不看完整本书得不到完整答案

**不合格的论点**：
- ❌ 一句话就能讲完的金句（没有知识缺口空间）
- ❌ 太学术，和观众日常无关（没有代入感）
- ❌ 大众本来就不相信的东西（没有刻板印象可破）

### Step 2：为每个论点选择最佳结构

| 论点特征 | 推荐结构 |
|----------|---------|
| 你自己曾经也搞错过 | A：困惑-书解 |
| 跟热点/日常话题直接相关 | B：场景代入 |
| 书里有一句特别炸裂的原话 | C：翻书实录 |

### Step 3：生成内容骨架

每条选题输出：

```markdown
### 选题 N：[标题]
- 结构：A / B / C
- 刻板印象：[要破的认知]
- 书的论点：[书怎么说的]
- 只给 30%：[你在内容里讲到哪里就停]
- 留给书的 70%：[观众需要自己去看的部分]
- 知识缺口钩子：[用什么话制造"未完成感"]
```

### Step 4：展开为完整内容

**30% 法则**（核心铁律）：
- 讲清楚「旧认知为什么错」— 这部分可以讲透
- 讲清楚「新框架长什么样」— 只讲入口/第一层
- **不要展开新框架的完整定义** — 这是书的工作
- 你的内容结束时，观众的状态应该是：「旧的我信不了了，新的我还没完全懂，我得自己去看」

## 输出格式

### 短视频口播稿

```markdown
## 选题 N：[标题]
结构：A / B / C
时长版本：3min / 5min / 10min

---

[完整口播文案]

---

预估时长：X 分钟
字数：X 字
知识缺口钩子：「……」
书中对应章节：第 X 章
```

**字数对照**：
- 3 分钟版：800-1000 字
- 5 分钟版：1500-2000 字
- 10 分钟版：3000-4000 字

### 图文帖子

```markdown
## 选题 N：[标题]
结构：A / B / C

### 封面文案
[1 句话，适合做封面大字]

### 正文
[800-1500 字]

### 标签建议
#标签1 #标签2 #标签3

---

书中对应章节：第 X 章
```

## 质量自检

每条内容过以下检查：

- [ ] 你是不是讲太透了？观众看完是"懂了"还是"想看书"？
- [ ] 你的角色是"受益者/好奇者"还是"专家/老师"？（必须是前者）
- [ ] 有没有明确的"未完成感"钩子？（"后面还有 N 层"、"这只是开头"）
- [ ] 刻板印象是不是大众真的相信的？（不是小众误解）
- [ ] 引书是否自然？（不能像广告）
- [ ] 30% 法则：内容是否只展开了框架入口，没有替书讲完？

---

## 批量模式

1. 先输出所有选题的骨架（标题 + 结构 + 30%/70% 划分）
2. 用户挑选要展开的
3. 逐条展开
