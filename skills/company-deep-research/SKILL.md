# 一手深度调研

> 不看研报结论，用一手信息源按维度拆解，交叉验证，输出可执行的调研报告。
> 目标时间：完整调研 ~20分钟，快速扫描 ~8分钟。

## 核心框架：七维分析模型

| 维度 | 核心问题 | 信息领先性 |
|------|---------|-----------|
| 1. 管理层决策质量 | 创始人在关键分叉口怎么选？ | 领先财报 3-5年 |
| 2. 市场结构裂缝 | 大厂"觉得没必要做"的地方在哪？ | 领先研报 1-2年 |
| 3. 技术周期位置 | 技术从实验室到量产走到哪一步了？ | 领先产品发布 1-3年 |
| 4. 产业链位置 | 公司在价值链的哪一层？是否被低估？ | 领先市场定价 |
| 5. 资本结构 | 谁在出钱？结构是否抗风险？ | 工商数据准实时 |
| 6. 隐含信号 | 招聘/专利/子公司在暗示什么？ | 领先公告 3-6月 |
| 7. 风险矩阵 | 如果我的判断错了，最可能错在哪？ | 反向验证 |

> 维度详解 → [references/dimensions-guide.md](references/dimensions-guide.md)
> 报告模板 → [references/report-template.md](references/report-template.md)

## 完整调研流程

### 输入解析

用户输入公司名或代码后，先确认：
1. 公司全称、股票代码、所属行业
2. 创始人/现任CEO姓名
3. 主营业务一句话描述

```bash
cd ~/.Codex/skills/MX_FinData && python3 scripts/get_data.py \
  --query "[公司名][代码] 基本信息 总市值 行业 主营业务 创始人 CEO"
```

---

### 第一批：并行数据采集（启动3个Agent，~5分钟）

#### Agent A：管理层画像（维度1+2）

**数据源：** Exa + WebSearch

```
Step A1 — 基础履历
  Exa people_search: "[创始人/CEO姓名]"
  WebSearch: "[创始人姓名] 履历 教育背景 创业经历"

Step A2 — 决策节点追踪
  WebSearch: "[创始人姓名] 演讲 采访 创业故事 关键决策"
  WebSearch: "[公司名] 创始人 转型 战略调整"
  Exa web_search: "[公司名] founder CEO interview strategy"

Step A3 — 承诺信号扫描
  WebSearch: "[创始人姓名] 增持 减持 股权变动"
  WebSearch: "[公司名] 高管持股 股权激励"
```

**提取要点：**
- 学历/职业路径（是否有行业深耕经验）
- 3个最关键的决策节点及结果
- 承诺密度信号（增持、零薪、全职投入等）
- 公开言论中的战略一致性（vs 实际行动是否匹配）

---

#### Agent B：财务数据全量拉取（维度5的数据准备）

**数据源：** MX_FinData

```
Step B1 — 核心财务
  "[公司名] 近3年营收 净利润 毛利率 净利率 ROE 经营现金流"

Step B2 — 运营效率
  "[公司名] 应收账款周转天数 存货周转天数 资产负债率 流动比率"

Step B3 — 资金面
  "[公司名] 前十大股东 质押比例 融资余额 解禁日期 回购 定增"

Step B4 — 估值参照
  "[公司名] PE PB PS 行业平均PE 近3年PE区间"
```

---

#### Agent C：产业链 + 技术路线（维度3+4）

**数据源：** WebSearch + Exa + MX_FinData

```
Step C1 — 产业链图谱
  WebSearch: "[公司名] 产业链 上下游 供应商 客户"
  WebSearch: "[行业名] 产业链图谱 竞争格局 2026"
  MX_FinData: "[公司名] 前五大客户 前五大供应商 营收构成 分部收入"

Step C2 — 技术路线
  WebSearch: "[公司名] 技术路线 研发方向 核心技术 专利"
  WebSearch: "[公司名] vs [主要竞争对手] 技术差异"
  Exa web_search: "[公司名] patent technology R&D innovation"

Step C3 — 竞争对手对标
  MX_FinData: "[竞争对手1] [竞争对手2] 营收 毛利率 市值 PE"
```

**提取要点：**
- 公司在价值链的具体位置（是否处于"被忽略的中间层"）
- 客户集中度（前5大客户占比 > 50% = 风险）
- 技术路线选择：跟随主流 vs 另辟蹊径
- 专利数量/质量趋势
- 竞争对手对标（谁在同一赛道，谁的毛利更高说明护城河更深）

---

### 第二批：并行深挖（~4分钟）

#### Agent D：隐含信号（维度6）

**数据源：** WebSearch

```
Step D1 — 招聘信号
  WebSearch: "[公司名] 招聘 最新岗位 2026"
  WebSearch: "[公司名] boss直聘 猎聘 岗位方向"
  → 招什么人 = 在布局什么方向（比产品发布早6-12个月）

Step D2 — 子公司/工商变更
  WebSearch: "[公司名] 子公司 新设 工商变更 2025 2026"
  WebSearch: "[公司名] 对外投资 参股 收购"
  → 新子公司 = 新赌注

Step D3 — 专利时间线
  WebSearch: "[公司名] 专利申请 最新专利 国家知识产权局"
  → 专利申请方向 = 未来1-3年的技术路线图
```

---

#### Agent E：财务交叉验证（维度5核心）

> 核心逻辑：不信公司自己说的数字，拿客户和供应商的数据反向验证。

**数据源：** MX_FinData

```
Step E1 — 正向验证：公司 → 客户
  "[主要客户1] 近3年采购成本 供应商变动"
  "[主要客户2] 近3年采购成本 供应商变动"
  → 客户的采购额增长 = 公司的营收真实

Step E2 — 反向验证：供应商 → 公司
  "[主要供应商] 近3年营收 客户集中度"
  → 供应商的营收和公司的采购额是否匹配

Step E3 — 现金流质量
  对比：净利润 vs 经营现金流
  → 经营现金流 < 净利润×0.7 连续2年 = 盈利质量差，标红
```

---

### 第三步：综合研判（~5分钟）

#### Phase 6：风险扫描 + 报告生成

```
Step 6.1 — 风险扫描
  MX_FinData: "[公司名] 解禁日期 质押比例 诉讼 处罚 审计意见"
  WebSearch: "[公司名] 风险 问题 争议 处罚 立案"
  WebSearch: "[公司名] 研报 评级 目标价 2026"

Step 6.2 — 七维评分（每维度1-5分）
  逐维度打分，参考 references/dimensions-guide.md 的评分标准

Step 6.3 — 生成报告
  按 references/report-template.md 格式输出完整报告

Step 6.4 — 反面论证（不可省略）
  回答："如果我的判断完全错了，最可能的原因是什么？"
  至少列出3个可能让投资论点失效的风险
```

## 铁律（不可违反）

| # | 规则 |
|---|------|
| 1 | **不信单一信息源**。每个关键判断至少两个独立来源交叉验证 |
| 2 | **反面论证不可省略**。每份报告必须回答"如果我错了" |
| 3 | **分开两件事**：好公司 ≠ 好股票。调研结论和估值判断分开输出 |
| 4 | **标注置信度**。每个维度的判断标注：高/中/低（数据充分/部分/不足） |
| 5 | **不编数据**。查不到的数据标"未获取"，不用模糊语言替代 |
| 6 | **客户/供应商必须交叉验证**。营收增长没有被客户数据验证的，标"待验证" |
| 7 | **招聘信号 > 公告**。公司说在做X但不招X方向的人 = 空话 |
| 8 | **创始人行动 > 创始人言论**。增持/减持/薪资变化比采访发言更真实 |
| 9 | **产业链位置决定利润归属**。不看绝对利润，看在价值链中的议价权 |
| 10 | **时间一致性是最强信号**。创始人10年做同一件事 > 每年换方向 |

## 与现有 Skill 的协作

```
company-deep-research  → 这家公司值不值得投？（定性，选股前）
       ↓
a-stock-analysis       → 什么价格买？买多少？（定量，买入时）
       ↓
mx-finance-data        → 底层数据接口（被两个 skill 调用）
mx-finance-search      → 消息面搜索
```

典型流程：
1. 用 `company-deep-research` 调研，确认值得投
2. 用 `a-stock-analysis` 模式一（买点分析）确认入场价格
3. 建仓后用 `a-stock-analysis` 模式二/三（盘中/预警）管理持仓
