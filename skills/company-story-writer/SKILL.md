# 公司叙事写作引擎·万字深度解读系列

> 系列名称固定为：**万字深度解读·[公司名]**
> 目标字数：**10000字以上**，不足万字视为未完成。
> 不写研报，写故事。让读者从头读到尾，每一章都有理由继续往下看。

---

## 核心理念

这个技能解决的不是"信息对不对"，而是"读者愿不愿意读下去"。

**与 company-deep-research 的关系：**
- company-deep-research → 事实底稿、数据、竞争格局（解决"值不值得投"）
- company-story-writer → 角度、结构、情绪张力、公众号表达（解决"值不值得讲，怎么讲"）

两者可以串联使用：先用 company-deep-research 调研，确认值得写，再用本技能转化为文章。

---

## 工作流程

### 阶段一：判断故事性（值不值得讲）

在开始写作前，先评估这家公司是否有足够的故事性。从以下 6 个维度打分（1-5 分）：

1. **人物张力** - 创始人/董事长是否有独特的人格特质、背景反差、或非典型路径
2. **命运转折** - 公司历史上是否有关键的生死时刻、战略转型、或重大赌注
3. **非共识下注** - 公司是否在做别人不看好但后来证明正确的事
4. **行业位置反差** - 公司在产业链中的位置是否被低估或误解
5. **争议性** - 市场对这家公司是否存在明显分歧
6. **当下性** - 为什么这家公司在"今天"值得被讲（踩中了什么周期/热点）

**输出：**
- 故事性评级：高（24-30分）/ 中（16-23分）/ 低（6-15分）
- 如果评级为"低"，向用户说明原因，建议是否继续
- 如果评级为"中"或"高"，推荐最适合的文章模型（见阶段三）

### ✋ Checkpoint ① · 方向确认

评级产出后，用 1-2 句话向用户确认：
> "这家公司故事性评级为 [X]，推荐从 [模型名] 入手。是否按这个方向抽骨架？或者你更想从别的角度切入？"

**用户可选三种回应：**
- (a) 按推荐方向 → 进入阶段二
- (b) 换一个模型 → 重新说明该模型的适用性后进入阶段二
- (c) 低评级还坚持写 → 明确告知风险（见「异常 3」），用户确认后进入阶段二

---

### 阶段二：抽取故事骨架

将调研中的事实信息转化为故事元素。这是从"冷信息"到"热结构"的关键转换。

> **先读 `references/angle-library.md`**，从 5 个维度（人物/公司/产品/行业/时间）里挑出 2-3 个最适合这家公司的切入角度，再开始填骨架卡。不要凭感觉定角度。

#### 2.1 基础转换表

| 事实元素 | 故事元素 | 提取要点 |
|---------|---------|---------|
| 董事长/创始人履历 | 人物原型 | 学历、职业路径、关键经历、人格特质 |
| 重组/并购/转型时间点 | 命运转折 | 背景、决策过程、结果、反事实推演 |
| 核心产品/技术 | 角色武器 | 独特性、护城河、与竞品差异 |
| 行业格局/竞争态势 | 外部战场 | 谁是对手、谁是盟友、战场规则 |
| 财务数据变化 | 战果验证 | 关键拐点、增长曲线、盈利质量 |
| 风险因素 | 悬念与反证 | 最大变量、可能出错的地方 |

#### 2.2 重点挖掘：8 类"留人信息"

这些信息能让读者继续往下看，是文章的"钩子"：

**1. 人格反差**
- 文科生去做硬科技
- 学者型董事长管理重工业
- 销售出身却押注技术
- 工程师出身却特别懂资本

**2. 关键转折**
- 差点卖掉公司
- 差点死掉但活下来了
- 一次重组翻身
- 一次赌注改变命运

**3. 管理气质**
- 集权型 vs 分权型
- 工程师文化 vs 销售文化
- 国企纪律感 vs 创业公司狼性
- 学院派慢热 vs 实战派激进

**4. 公司气质类比**
- "像钨行业的紫金矿业"
- "像一个央企版的极致制造工厂"
- "像资源股里混进来的一家设备公司"

**5. 争议点**
- 高估值 vs 低成长
- 央企体制是否拖累效率
- 技术真突破还是故事包装
- 行业景气是周期还是结构

**6. 关系网**
- 创始人与地方政府的关系
- 公司与大客户的依赖/博弈
- 公司与实控人体系的关联
- 公司与同行的分工/竞争

**7. 组织风格**
- 喜欢并购还是内生增长
- 喜欢做全链条还是只做一环
- 喜欢讲远景还是讲执行
- 激进扩张还是稳健经营

**8. 时代感**
- 这家公司为什么在"今天"才值得被讲
- 它到底踩中了哪一波大周期
- 它的故事与当下热点的关联

#### 2.3 阶段二输出格式（结构化骨架卡）

抽取完骨架后，用固定模板输出，不要直接进入阶段三。

```
## 故事骨架卡

**一句话内核**：[≤30 字，点明"这家公司为什么值得讲"]

**人物原型**：[谁 + 最有戏的一个细节]
**命运转折**（按时间轴）：
1. [年份·事件·非共识点]
2. [年份·事件·非共识点]
3. [年份·事件·非共识点]

**角色武器**：[1-2 句讲清护城河]
**外部战场**：[对手是谁 + 战场规则是什么]
**战果验证**：[最亮眼的 1-2 个财务拐点]
**悬念与反证**：[最大的变量 / 市场最大的分歧点]

**留人信息清单**：[从 8 类里勾出用得上的 3-5 条]
**推荐叙事模型**：[A/B/C/D/E + 理由一句话]
```

### ✋ Checkpoint ② · 骨架确认

骨架卡输出后，明确问用户：
> "这个骨架对不对？有没有哪条需要调整或补充？确认后我生成选题包（标题+开头钩子+结构）。"

**用户可选：**
- (a) 骨架 OK → 进入阶段三
- (b) 补人物细节 / 补某个节点 → 回阶段二补完再来
- (c) 换内核 → 重写一句话内核再继续

---

### 阶段三：生成选题包

不要直接写正文，先给用户一组可选角度。这一步是为了确保方向对了再动笔。

**输出内容：**

1. **3 个标题候选**
   - 每个标题都要有钩子（反差、判断、悬念）
   - 避免"深度分析""研究报告"这类死词
   - 优先用：
     - "为什么……"
     - "真正……的是……"
     - "他赌了三次"
     - "这家公司最值钱的不是……"
     - "很多人没看懂……"

2. **3 个开头钩子**
   - 每个钩子都是文章的前 2-3 段
   - 必须包含一个反差或判断
   - 让读者产生"咦，有意思"的感觉
   - **先读 `references/hook-patterns.md`**，从 6 类钩子模式（认知反转 / 对比冲击 / 数据冲击 / 悬念设置 / 场景切入 / 反问开场）里选择最匹配的 3 种，各产出一个具体钩子。不要凭空写。

3. **2 种文章结构方案**
   - 基于 5 种叙事模型（见 references/narrative-models.md）
   - 说明为什么推荐这个结构
   - 列出每个部分的核心内容

4. **1 句核心判断**
   - 这篇文章最想传达的观点是什么
   - 用一句话说清楚

5. **篇幅说明**
   - 本系列固定为**万字深度解读**，目标 10000-15000 字
   - 字数不足万字不输出，继续补充直到达标
   - 万字不是"写多"，而是"把每个维度都讲透"

**等待用户确认后再进入阶段四。**

### ✋ Checkpoint ③ · 选题确认

选题包输出后：
> "这三个标题+三个钩子里，哪个方向你最想要？也可以指出某几个要素想要混合，我再合成最终方案。"

**硬规则**：用户没明确选定前，不得开始写正文。这个检查点是防止方向跑偏后浪费万字产出的最后屏障。

---

### 阶段四：撰写正文

**篇幅铁律：目标 10000-15000 字。写完后必须自查字数，不足万字必须继续补充，不得输出未完成的文章。**

万字的内容从哪来？不是靠"凑字"，而是把每个维度真正讲透：

| 章节 | 建议字数 | 核心任务 |
|------|----------|----------|
| 引言（悬念/反差开场） | 500-800字 | 抓住读者，提出核心问题 |
| 公司/行业背景 | 800-1200字 | 讲清楚这个行业是怎么运作的，普通读者能听懂 |
| 人物：掌舵者是谁 | 800-1200字 | 管理层的背景、性格、决策风格 |
| 命运转折（核心叙事） | 2000-3000字 | 关键战略决策的来龙去脉，分多个节点讲 |
| 产品/技术/竞争力 | 1500-2000字 | 公司真正的护城河在哪，与对手的差距 |
| 财务解读 | 1000-1500字 | 数据融入叙事，讲清楚钱是怎么赚的 |
| 当下时机 | 800-1000字 | 为什么现在值得讲，踩中了什么周期 |
| 风险与争议 | 800-1000字 | 最大的变量是什么，市场在哪里可能错了 |
| 结尾判断 | 500-800字 | 有态度的结论，留下"未完待续"感 |

**总计：约 8700-12500 字，加上过渡段落和细节展开，达到万字以上。**

根据用户选定的标题、结构，按以上章节框架撰写完整文章。

#### 4.0 字数预算簿（开写前必做）

动笔前先输出一个字数预算表给自己，按章节目标字数做分配。每写完一章立即报告实际字数。

```
[字数预算簿]
引言：目标 600 | 实际 ___ | 差值 ___
背景：目标 1000 | 实际 ___ | 差值 ___
人物：目标 1000 | 实际 ___ | 差值 ___
命运转折：目标 2500 | 实际 ___ | 差值 ___
产品/技术：目标 1800 | 实际 ___ | 差值 ___
财务解读：目标 1200 | 实际 ___ | 差值 ___
当下时机：目标 900 | 实际 ___ | 差值 ___
风险争议：目标 900 | 实际 ___ | 差值 ___
结尾判断：目标 600 | 实际 ___ | 差值 ___
—————————————————
累计目标：10500 | 累计实际：___
```

**规则**：某章差值累计超过 ±200 字时立即调整后续章节预算，避免最后总量偏离太远。这个预算簿可以不在文章正文里展示，但必须在内部执行。

#### 4.1 公众号写作七技巧（速查）

> **核心原则**：每 300-500 字必须有一个"钩子"，让读者继续往下读。
> **写作前先读 `references/writing-techniques.md` 获取每条技巧的 ✅/❌ 示例**。

| # | 技巧 | 一句话抓手 |
|---|------|-----------|
| 1 | 开头反差/悬念 | 前 3 段必须让读者产生"咦？" |
| 2 | 问题驱动 | 每大段抛问题 → 给答案 → 引下一个问题 |
| 3 | 时间锚点 | 用年份切章，每个时间点都是小高潮 |
| 4 | 数据说人话 | 数据融入叙事，不堆表格 |
| 5 | 每 500 字一个小高潮 | 反转 / 判断 / 类比 / 反问 |
| 6 | 章节三段式 | 抛问题 → 展开（≤3 条） → 给结论 |
| 7 | 结尾留悬念 | 不要"综上所述"，要"未完待续" |

**行文细节**：每段 3-5 句、≤150 字；多用句号少用逗号；避免"综上所述/总而言之"；每大段必须推动问题。

#### 4.2 叙事模型选择

根据公司特点和用户需求，选择最合适的叙事模型。详细结构见 `references/narrative-models.md`。

**快速选择指南：**
- 创始人/董事长有强烈个人特质 → **模型 A：人物下注型**
- 老公司重生、国企转型、借壳翻身 → **模型 B：公司命运型**
- 公司走的是行业偏门、非主流路线 → **模型 C：行业裂缝型**
- 公司有独特的管理风格或组织文化 → **模型 D：公司气质型**
- 市场对公司存在明显分歧 → **模型 E：争议拆解型**

#### 4.3 开头句式库

如果不知道怎么开头，可以参考 `references/hook-patterns.md` 中的句式模板。

## 数据来源与工具

### 优先使用 company-deep-research 的输出

如果用户已经用 company-deep-research 做过调研，直接读取其输出作为事实底稿。

### 如果没有现成调研，按需获取数据

根据文章需要，调用以下工具：

1. **基础信息** - MX_FinData
   - 公司全称、股票代码、行业、主营业务
   - 创始人/CEO 姓名、持股比例

2. **管理层信息** - WebSearch + Exa
   - 创始人/CEO 履历、教育背景、创业经历
   - 公开演讲、采访、关键决策
   - 增持/减持、股权变动

3. **财务数据** - MX_FinData
   - 近 3 年营收、净利润、毛利率、ROE
   - 关键财务拐点
   - 前十大股东、质押比例

4. **产业链与竞争** - WebSearch + MX_FinData
   - 上下游关系、主要客户和供应商
   - 竞争对手对标
   - 行业格局

5. **争议与风险** - WebSearch + MX_FinData
   - 市场争议点
   - 风险因素
   - 研报观点分歧

**原则：只获取写文章必需的数据，不做全量调研。**

## 异常与边界处理

### 情况 1：调研信息不全（缺人物/财务/争议点）

**判定**：阶段一故事性评分时，发现某维度没有事实支撑。

**处理**：
1. **不编造**。缺什么就说明缺什么，不要靠"行业常识"推断人物性格或管理风格。
2. 列出「已知信息」和「写作必需但缺失的信息」两张清单给用户。
3. 让用户选择：
   - (a) 补查：调用 MX_FinData / WebSearch / Exa 针对缺项补查（最多 3 次）
   - (b) 改角度：缺人物就不写人物下注型，改用行业裂缝型或公司命运型
   - (c) 降级：接受信息不全，产出 6000-8000 字中篇而非万字长文
4. 用户不选就默认 (b) 改角度。

### 情况 2：MX_FinData / 数据源查不到

**触发**：调用 MX Skills 返回空、超时或报错。

**处理**（按 MEMORY.md 的金融数据规则）：
1. 先打印 raw JSON，确认是真 API 错误还是 status=0 被误判。
2. 重试一次。
3. 仍失败 → 降级 WebSearch + Exa 补数据，在文章末尾注明「财务数据来源：公开研报/公告」。
4. 如果连公开资料都没有（新股/港股/美股个别冷门股），明确告诉用户「此公司数据稀薄，建议换标的或只写人物/行业角度」。

### 情况 3：低故事性 / 敏感行业 / 高风险标的

**触发**：
- 故事性评分 ≤ 15 分（低）
- 行业属于：赌博、高利贷、涉讼退市、ST 股、被监管立案公司
- 创始人存在未结诉讼、刑事案件

**处理**：
1. **不直接拒绝**，但必须先告知风险，让用户确认。
2. 给出三个选项：
   - (a) 换角度写行业而不写公司（如「某退市 ST 股背后的行业隐喻」）
   - (b) 改成「反面案例」叙事，把争议当主线（模型 E 争议拆解型）
   - (c) 放弃这个标的，推荐同行业替代标的
3. 一律不写带有"值得买入""强烈推荐"等投资建议的语句——本技能产出内容性文章，不是投资报告。

### 情况 4：万字拉不满 / 字数不达标

**触发**：阶段四写完，统计字数 < 10000。

**处理**（按优先级递进）：
1. **先查是哪一章不够**（按章节字数表自查），不要平均稀释。
2. 针对字数不足的章节，选一种加深手法：
   - **命运转折章不足** → 补"反事实推演"（如果当年没这么做会怎样）
   - **产品/技术章不足** → 加一个具体产品的制造/研发场景细节
   - **财务章不足** → 加一张同行对比（三家公司关键指标横向对比）
   - **人物章不足** → 补一段公开采访/演讲金句 + 其决策的思维印记
   - **风险章不足** → 补「最强看空观点」+ 反驳
3. 每次只补一处，补完重新数字数，不要一次性膨胀。
4. 如果补了两轮仍不到万字，说明这家公司素材不足，主动降级为中篇（6000-8000 字）并告知用户，不要硬凑。

**禁止行为**：
- ❌ 用排比句、形容词堆叠凑字数
- ❌ 把同一观点换说法重复多次
- ❌ 大段引用财报原文、研报原文充数

## 输出格式

### 阶段一输出：故事性评估

```
## 故事性评估

**评分：**
- 人物张力：4/5
- 命运转折：5/5
- 非共识下注：3/5
```

### 阶段五（可选）：配图协作 · baoyu-article-illustrator

触发条件：Checkpoint ④ 后用户选择 (a)。

本阶段不是自己配图，而是**把准备好的上下文传给 baoyu-article-illustrator**。公司叙事文章的章节特征已经预埋在下面的默认方案里，直接套用即可。

#### 5.1 万字深度解读专用配图方案

| 参数 | 默认值 | 备注 |
|------|--------|------|
| type | **mixed**（timeline + infographic + scene + comparison） | 单一 type 吃不下公司叙事的多层次 |
| density | **balanced（4-6 张）** | 万字文章 3 张起步，超过 6 张干扰阅读 |
| style | **elegant**（默认）/ notion / warm | 按公司气质选（见 5.2） |

#### 5.2 Style 决策树

```
公司性质
├── 央国企 / 大型制造 / 金融 / 传统蓝筹 → elegant
├── 科技 / SaaS / 互联网 / 新能源 → notion 或 blueprint
├── 消费品 / 生活方式 / 文创 → warm 或 watercolor
└── 创始人传记色彩重 / 人物故事为主 → warm 或 editorial
```

#### 5.3 章节-配图映射（推荐位点）

不是每章都配，按**叙事高潮点**挑选：

| 章节 | 推荐 Type | 图的作用 |
|------|-----------|----------|
| 引言 | scene / infographic | 开场画面感 或 核心数据冲击（必配） |
| 命运转折 | **timeline** | 多次下注的时间轴（万字深度解读的核心配图，必配） |
| 产品/技术 | framework / infographic | 护城河拆解 或 产业链位置（推荐） |
| 财务解读 | infographic | 关键拐点可视化（推荐） |
| 风险争议 | comparison | 多空观点对比（可选） |

**不配图的章节**：人物章（避免生成失真的人脸）、结尾判断（文字更有力）。

#### 5.4 调用方式

向 baoyu-article-illustrator 移交时，提供以下上下文：

```
请用 baoyu-article-illustrator 为以下文章配图：

**文章路径**：[文件路径 或 刚生成的完整正文]
**推荐参数**：
  - type: mixed
  - density: balanced
  - style: [按 5.2 决策树]
**章节配图指引**：[按 5.3 勾选的位点列出，格式："第 N 章「章节名」用 [type]，表达：[visual content]"]
**禁止事项**：
  - 人物章不生成带人脸的图
  - 金融数据图不得出现"买入/卖出/目标价"等投资建议文字
  - 不要把比喻（如"钨产业的紫金矿业"）字面化
```

#### 5.5 配图完成后的校验

baoyu 输出后，回到本 skill 做最后一步检查：
- [ ] 图片插入位置是否匹配章节叙事节奏
- [ ] 财务/数据图是否与正文数字一致（不一致就替换或删除）
- [ ] 整体风格是否匹配公司气质（不匹配就换 style 重跑）
- [ ] 图片 alt 文本是否为中文（公众号发布需要）

## 铁律

1. **不是写研报，是写故事** - 如果读起来像研报，就失败了
2. **不堆术语，不堆表格** - 数据要融入叙事，不要单独列表
3. **每个判断都要有故事支撑** - 不要空泛地说"这家公司很厉害"，要讲它怎么厉害的
4. **结尾必须有明确判断** - 不要模棱两可，要有态度
5. **标题必须有钩子** - 如果标题平淡无奇，文章再好也没人点开
