# 对话实时整理系统 (Conversation Notes)

本 Skill 在与用户对话的同时，将对话内容实时整理到结构化的 Markdown 文档中，实现「对话即沉淀」的工作流。

## ⚡ 自动激活模式

**此 Skill 默认自动激活**，每次新对话开始时：

1. **立即创建文档**：在配置的路径下创建当天的笔记文件
2. **加载长期记忆**：同时读取 `~/.Codex/MEMORY.md` 和项目级 `LESSONS.md`（如果存在），作为内部上下文影响后续回答（与 reflect v2 联动）
3. **实时更新**：对话过程中持续更新文档内容
4. **无需触发词**：不需要说「开始记录」，自动运行

**停用方式**：
- 说「暂停记录」或 `/notes pause`
- 在 AGENTS.md 中设置 `auto_notes: false`

---

## 首次对话行为

**当检测到新对话开始时**，Codex 应该：

1. 检查笔记目录是否存在，不存在则创建
2. 创建新的笔记文件（格式：`YYYY-MM-DD-HH-mm-主题.md`）
3. 在回复的末尾简短提示：`📝 笔记已创建：[文件路径]`
4. 后续回复中，如有重要内容更新，提示：`📝 已记录`

**文件路径优先级**：
1. 当前工作目录的 `./notes/` 文件夹（默认）
2. 全局配置的 `notes_path`（如果设置了）

**关键词触发：**
- 用户说「开始记录」「帮我记录」「整理笔记」
- 用户使用 `/notes` 命令
- 用户明确要求将对话内容整理成文档

## 阶段一：初始化

### 1.1 询问配置（首次使用）

首次触发时，询问用户：

1. **文档存放位置**：
   - Obsidian Vault 路径（如 `~/Obsidian/Vault/Codex-Sessions/`）
   - 当前工作目录的 `notes/` 子文件夹
   - 自定义路径

2. **文档命名方式**：
   - 日期 + 主题：`2025-02-04-项目讨论.md`
   - 仅主题：`项目讨论.md`
   - 时间戳：`1707062400-session.md`

3. **Obsidian 特性**：
   - 是否使用双向链接 `[[概念]]`
   - 是否自动添加标签 `#tag`

如用户已在 AGENTS.md 中配置，跳过询问。

### 1.2 创建/打开文档

```markdown
# [对话主题]

> 创建时间：YYYY-MM-DD HH:mm
> 标签：#Codex-session #[自动识别的主题标签]

---

## 背景与目标

[用户描述的背景或目标，首次交流后填写]

---

## 讨论记录

[按时间线记录讨论要点]

---

## 关键结论

[汇总所有重要结论]

---

## 待办事项

- [ ] [从对话中提取的待办]

---

## 开放问题

[尚未解决的问题]

---

## 参考链接

[对话中提到的链接、文件路径等]
```

## 阶段二：实时整理（核心）

### 2.1 整理触发时机

**每次回复时评估**：
- 是否有新的关键结论？→ 更新「关键结论」
- 是否有新的待办事项？→ 更新「待办事项」
- 是否讨论了重要概念？→ 更新「讨论记录」+ 可能添加双向链接
- 是否有未解决的问题？→ 更新「开放问题」
- 是否提到了链接/文件？→ 更新「参考链接」

**不更新的情况**：
- 简单的确认性回复（"好的"、"明白了"）
- 纯代码输出（除非用户要求记录）
- 闲聊内容

### 2.2 更新策略

**使用 Edit 工具进行增量更新**，而非每次重写整个文档：

```
# 追加讨论记录
Edit: 在「## 讨论记录」下追加新条目

### HH:mm - [小标题]
- 要点1
- 要点2
- 相关概念：[[概念A]]、[[概念B]]
```

```
# 追加关键结论
Edit: 在「## 关键结论」下追加

- **[结论标题]**：[结论内容] `[HH:mm]`
```

```
# 追加待办事项
Edit: 在「## 待办事项」下追加

- [ ] [待办内容] `[来源：HH:mm 讨论]`
```

### 2.3 智能提取规则

**关键结论识别**：
- 用户说「所以...」「那就这样定了」「结论是」
- Codex 总结性回复「综上所述」「建议采用」
- 明确的决策点

**待办事项识别**：
- 「我需要...」「待会要...」「别忘了...」
- 「下一步是...」「TODO」
- 明确的行动项

**概念双向链接**：
- 专业术语、技术名词
- 人名、公司名、项目名
- 用户自定义的关键词列表

## 阶段三：对话结束

### 3.1 整理完善

对话即将结束时（用户说「就这样」「先这样」「我去忙了」等）：

1. **补充背景**：如果「背景与目标」为空，根据对话内容补充
2. **整理结论**：检查是否有遗漏的重要结论
3. **确认待办**：向用户确认待办事项列表
4. **添加标签**：根据讨论内容自动添加 2-5 个相关标签

### 3.2 生成摘要

在文档开头添加：

```markdown
## 本次对话摘要

> **主题**：[一句话概括]
> **时长**：约 [X] 分钟
> **产出**：[N] 条结论，[M] 项待办
>
> **核心收获**：
> 1. [最重要的收获1]
> 2. [最重要的收获2]
> 3. [最重要的收获3]
```

### 3.3 快速反思（与 reflect v2 联动）

**在生成摘要后、导出选项前**，自动执行一次快速反思：

1. 回顾整个对话，寻找最值得带走的**一个认知更新**（不是摘要，不是结论列表，而是"如果只能带走一个想法，是什么？"）
2. 用 2-3 句自然语言输出，风格像在和朋友复盘：
   ```
   💭 反思：[叙事性的洞见，锚定对话中的具体时刻]
   → 说"持久化"写入知识库，说"/reflect"进入完整复盘
   ```
3. 在当前 session notes 文件末尾追加 `## 💭 Quick Reflection` 区域
4. 如果用户确认持久化，按照 reflect v2 的持久化流程写入 `~/.Codex/MEMORY.md`
5. **v2 新增**：如果 MEMORY.md 中有相关旧经验被再次验证，同步更新引用计数和验证等级

**注意**：快速反思不是必须的。如果这次对话确实没有什么深层收获（纯执行任务、简单问答等），可以跳过，不要硬凑。

### 3.5 v2 新增：智能反思提示

当检测到以下模式时，在笔记整理后主动建议完整反思：
- 对话中发生过方向性调整（转折点）
- 用户表达了"终于搞定了"类的感叹
- 对话超过 30 分钟或 20 轮以上
- 涉及复杂决策或多方案权衡

提示方式（轻量一行）：
```
💭 这次过程挺值得复盘的，要 `/reflect` 一下吗？
```

### 3.4 导出选项

询问用户是否需要：
- **思维导图**：生成 Markmap 格式（可在 Obsidian 中渲染）
- **行动清单**：仅导出待办事项
- **分享版本**：去除内部备注，生成可分享的版本

## 配置选项

用户可在项目 AGENTS.md 或全局 AGENTS.md 中配置：

```markdown
# Conversation Notes 配置

## 文档位置
notes_path: ~/Obsidian/Vault/Codex-Sessions/

## 命名规则
naming: date-topic  # date-topic | topic-only | timestamp

## Obsidian 特性
obsidian:
  backlinks: true      # 双向链接
  tags: true           # 自动标签
  frontmatter: true    # YAML frontmatter

## 自动整理
auto_notes: false      # 是否自动开始记录（无需触发词）

## 关键词列表（用于双向链接）
keywords:
  - Codex
  - LLM
  - [其他自定义关键词]

## 更新频率
update_frequency: smart  # smart | every-reply | manual
```

## 快捷命令

| 命令 | 说明 |
|------|------|
| `/notes` 或 `/notes start` | 开始记录当前对话 |
| `/notes stop` | 停止记录 |
| `/notes status` | 查看当前记录状态 |
| `/notes add [内容]` | 手动添加一条笔记 |
| `/notes todo [内容]` | 手动添加一条待办 |
| `/notes summary` | 生成当前对话摘要 |
| `/notes export mindmap` | 导出为思维导图格式 |
| `/notes export todo` | 仅导出待办事项 |
| `/notes config` | 修改配置 |

## 最佳实践

1. **在 Obsidian Vault 目录启动 Codex**
   这样所有笔记自动保存到你的知识库

2. **配合 Obsidian 的实时预览**
   打开文档后，Obsidian 会自动刷新显示更新

3. **定期 review 双向链接**
   通过 Obsidian 的图谱视图发现概念间的关联

4. **利用标签分类**
   `#Codex-session` 标记所有 Codex 对话
   `#actionable` 标记有待办事项的笔记

5. **建立模板库**
   为不同类型的对话创建不同模板（产品讨论、学习笔记、问题排查等）

## 技术说明

### 为什么用 Edit 而非 Write？

- **增量更新**：Edit 工具只修改需要变化的部分
- **保留格式**：不会意外覆盖用户手动添加的内容
- **实时可见**：Obsidian 能立即看到每次小更新

### 文档为什么能「实时」更新？

1. Codex 使用 Edit 工具写入文件
2. 文件系统触发变更事件
3. Obsidian 监听到变更，自动刷新视图

整个过程通常在 1 秒内完成。

## 常见问题

**Q: 可以同时记录多个对话吗？**
A: 可以，每个对话会创建独立的文档。

**Q: 如何防止敏感信息被记录？**
A: 说「这条不用记录」或使用 `/notes pause` 暂停。

**Q: 文档太长怎么办？**
A: 长对话会自动按主题分段，也可以说「新建一个文档继续」。

**Q: 如何修改已记录的内容？**
A: 直接在 Obsidian 中编辑，或说「帮我修改刚才记录的 XXX」。
