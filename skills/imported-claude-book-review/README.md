# 书评生成 Agent Skill v2.0（融合人性化写作）

这是一个升级版的 Claude Code Agent Skill，专门用于生成**真实、有灵魂、避免 AI 腔调**的书评。

## ✨ v2.0 新特性

融合了 [Humanizer-zh](https://github.com/op7418/Humanizer-zh) 的核心原则，生成的书评：

- ✅ 像真人写的，有观点、有态度、有个性
- ✅ 避免 AI 高频词汇（"此外"、"至关重要"、"充满活力"等）
- ✅ 打破公式化结构（不用三段式、否定排比等）
- ✅ 变化句子节奏，有短促句也有长句
- ✅ 适当使用第一人称，表达真实感受
- ✅ 承认复杂性和不确定性
- ✅ 对感受具体化（不说"引人入胜"，说"凌晨两点还在翻"）

## 📥 安装状态

已安装在：
```
~/.config/claude/skills/book-review/SKILL.md
~/.config/claude/skills/humanizer-zh.md
```

重启 Claude Code 后生效。

## 🎯 核心改进

### 1. 避免 AI 痕迹

**不再使用**：
- ❌ "此外"、"然而"、"至关重要"等 AI 高频词
- ❌ "这不仅……而且……"式否定排比
- ❌ 三段式列举（现在用两项或四项）
- ❌ "作为……的证明"、"标志着"等夸大表述
- ❌ "充满活力"、"丰富的"、"深刻的"等宣传词
- ❌ 表情符号装饰标题

### 2. 增加真实感

**现在会**：
- ✅ 变化句子长度和节奏
- ✅ 表达真实观点和态度
- ✅ 承认不确定性："我不太确定"、"可能是这样"
- ✅ 使用第一人称："读到这里我开始怀疑"
- ✅ 具体化感受："凌晨两点还在翻"而非"引人入胜"
- ✅ 允许跑题和半成型想法

### 3. 信任读者

- 直接说重点，不绕圈
- 删除填充短语
- 用简单系动词（"是"而非"作为/充当"）
- 不过度限定

## 📋 生成的书评结构

虽然有结构，但不会机械执行：

1. **基本信息** - 简单直接
2. **内容概述** - 说清楚书讲什么，不用套话
3. **真实感受** - 有观点、有态度
4. **具体优点** - 用具体例子，不堆形容词
5. **真实批评** - 指出真正的问题
6. **推荐建议** - 直接说适合谁、不适合谁

## 🛠️ 可用风格

### 自然对话风格（默认推荐）
像跟朋友聊天，用第一人称，可以跑题，承认不确定性。

### 学术分析风格
即使是学术风格，也避免 AI 腔调。引用具体，可以有观点。

### 快速判断风格
简短直接，给出结论和理由，不用标题式列表。

## 🐛 故障排除

### Skill 没有使用人性化风格？

1. 确认 humanizer-zh.md 已安装在 `~/.config/claude/skills/`
2. 确认 book-review 的 version 是 2.0.0
3. 重启 Claude Code
4. 明确告诉 Claude："用人性化的语气写"

### 书评太口语化/太随意？

Skill 设计目标是"像真人写的"，不是"随意"。如果觉得太口语化，可以在请求时说明："写一个专业但自然的书评"。
