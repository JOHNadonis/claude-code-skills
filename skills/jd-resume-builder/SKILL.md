# JD 简历生成器（jd-resume-builder）

基于 JD 驱动的智能简历定制工具。核心理念：**一份经历库 + 一份 JD = 一份高度匹配的简历**。

输出格式兼容 [木及简历](https://www.mujicv.com)（react-resume-site），可直接粘贴预览和导出 PDF。

---

## 触发方式

```
/jd-resume-builder [可选：JD文件路径]
```

**自然语言触发**：
- 帮我根据这个 JD 写简历
- 根据岗位要求生成简历
- 简历匹配
- 针对这个岗位定制简历

---

## 执行流程

### ⚡ 总览

```
Phase 0: 初始化 → 加载个人经历库
Phase 1: 获取 JD → 自动识别输入方式
Phase 2: JD 分析 → 提取关键词和要求
Phase 3: 匹配分析 → 从库中筛选最相关内容
Phase 4: 分节生成 → 每节等待用户确认
Phase 5: 最终输出 → 完整 Markdown（mujicv.com 格式）
```

---

### Phase 0：初始化 — 加载个人经历库

**步骤**：
1. 尝试读取 `~/.Codex/personal-profile.md`
2. **如果文件存在**：静默加载，输出一行确认：`✅ 已加载个人经历库（X 段工作经历，Y 个项目，Z 项技能）`
3. **如果文件不存在**：进入建库流程

**建库流程**（三选一，自动识别用户意图）：

**方式 A — 已有简历文件**：
> 如果用户提供了 `.md` 文件路径，使用 Read 工具读取，转换为经历库格式保存到 `~/.Codex/personal-profile.md`

**方式 B — 自由文本粘贴**：
> 如果用户粘贴了一段个人经历文字，解析并结构化后保存为 `~/.Codex/personal-profile.md`

**方式 C — 结构化问答引导**：
> 如果用户没有现成材料，按以下顺序收集信息（每次问 1-2 个问题，不要一次性列出所有问题）：
> 1. 基础信息（姓名、联系方式、城市）
> 2. 教育背景（学校、专业、时间、亮点）
> 3. 工作经历（按时间倒序，每段工作的公司、职位、时间、主要成就）
> 4. 项目经历（项目名、背景、技术栈、你的角色、成果）
> 5. 技能全集
> 收集完成后，将信息写入 `~/.Codex/personal-profile.md`（使用模板：`~/.Codex/skills/jd-resume-builder/templates/personal-profile-template.md`）

建库完成后提示：
```
✅ 个人经历库已创建：~/.Codex/personal-profile.md
💡 可以随时编辑此文件添加新项目和经历，下次使用时自动加载。
```

### Phase 1：获取 JD

**自动识别输入方式**：

| 用户提供的内容 | 处理方式 |
|--------------|----------|
| 文件路径（如 `~/jd.md`）| 用 Read 工具读取 |
| URL | 用 WebFetch 工具获取 |
| 直接粘贴的文本 | 直接使用 |
| 无内容（只说了触发词）| 提示用户粘贴 JD 或提供文件路径 |

输出简短确认：`✅ 已获取 JD（岗位：[岗位名称]，[字数]字）`

---

### Phase 2：JD 深度分析

对 JD 进行结构化解析，输出分析报告：

```
## 📋 JD 分析报告

**目标岗位**：[岗位名称]
**招聘公司**：[公司名，如有]
**经验要求**：[年限要求]
**学历要求**：[学历]

### 🎯 必须具备的能力/技能
- [技能1]（来源原文："..."）
- [技能2]

### ⭐ 加分项/优先项
- [加分技能1]
- [加分技能2]

### 📌 核心职责关键词
[提取的关键词，以逗号分隔]

### 🔍 软实力/文化信号词
[提取的关键词，以逗号分隔]
```

输出后询问用户：
> `分析正确吗？有补充或修正的地方吗？（回复"继续"进入下一步，或告诉我修正点）`

---

### Phase 3：匹配分析

基于 Phase 2 的分析结果，从个人经历库中筛选内容：

**匹配逻辑**：
1. **技能匹配**：将 JD 技能需求与经历库技能全集做交叉对比
2. **项目匹配**：用 JD 关键词与每个项目的「关键词标签」做语义匹配，按相关度排序
3. **工作经历匹配**：每段工作经历与 JD 职责的重叠程度

输出推荐入选清单：

```
## 🔗 匹配推荐清单

### 推荐入选的工作经历
✅ [公司A] [职位] — 匹配理由：[包含 X、Y 等 JD 关键技能]
⚠️ [公司B] [职位] — 部分匹配（建议作为补充）

### 推荐入选的项目（按匹配度排序）
1. ⭐⭐⭐ [项目A] — 高度匹配（涵盖 JD 核心技能：X、Y、Z）
2. ⭐⭐ [项目B] — 中度匹配
3. ⭐ [项目C] — 轻度匹配（建议选择前2个即可）

### 技能展示建议
重点突出：[技能列表，按 JD 重要性排序]
可提及：[次要技能]
建议省略：[不相关技能]
```

询问用户：
> `你是否同意以上选择？可以指定增减某个项目或工作经历，或直接回复"开始生成"。`

---

### Phase 4：分节交互生成

**重要**：每节单独生成并输出，等待用户回复「继续」或「修改：...」后再生成下一节。**不要**一次性输出所有节。

---

#### 4.1 生成头部信息

输出格式：
```markdown
# [姓名] - [目标岗位名称（与JD岗位高度一致）]

::::: header
:::: title
## 个人信息
::::

::: left
- [icon:phone [手机号]]
- [icon:email [邮箱]](mailto:[邮箱])
- 所在地：[城市]
- 求职状态：在职 / 离职
:::

::: right
- [icon:github GitHub]([链接])（如有）
- [icon:blog 个人博客]([链接])（如有）
:::
:::::
```

提示：
> `📌 头部信息已生成。回复"继续"生成个人简介，或告诉我需要修改的地方。`

---

#### 4.2 生成个人简介

**写作指引**：
- 长度：3-5 句话，150-250 字
- 结构：[年限] + [核心方向] + [代表成就] + [JD 对应能力] + [软实力]
- **必须**自然嵌入 JD 提取的 3-5 个关键词
- 使用第一人称，专业但不官僚

输出格式：
```markdown
## 个人简介

[生成的简介文字]
```

提示：
> `📌 个人简介已生成（已嵌入关键词：[关键词列表]）。回复"继续"生成工作经历，或告诉我修改点。`

---

#### 4.3 生成工作经历

**写作指引**：
- 每条 bullet point 以动词开头（负责、主导、优化、设计、实施...）
- 优先包含量化数据（减少 X%、提升 Y 倍、管理 Z 人团队）
- 每条 bullet 自然嵌入 1-2 个 JD 关键词
- 每段工作写 3-5 条 bullet，聚焦与 JD 最相关的成就

输出格式：
```markdown
## 工作经历

### [公司名称] | [职位] | [YYYY.MM] - [YYYY.MM / 至今]

- [成就，含量化指标和JD关键词]
- [成就，含量化指标和JD关键词]
- [成就，含量化指标和JD关键词]

### [公司名称2] | [职位] | [时间段]

- ...
```

提示：
> `📌 工作经历已生成。回复"继续"生成项目经历，或指定要修改的公司/条目。`

---

#### 4.4 生成项目经历

**写作指引**：
- 选取 Phase 3 推荐的前 2-3 个项目
- 每个项目：1 句背景描述 + 2-4 条成就 bullet
- 技术标签使用反引号格式（`` `技术名` ``），放在标题行
- 成就优先量化（处理 X 万条数据、降低 Y% 延迟等）
- **项目名称**：如果项目名称不够抓眼，可适当优化表述

输出格式：
```markdown
## 项目经历

### [项目名称] `技术1` `技术2` `技术3`

> [一句话描述：这是什么项目，解决什么问题，我的角色]

- [核心职责/成就，含量化指标]
- [技术难点及解决方案]
- [业务价值/成果]

### [项目名称2] `技术A` `技术B`

> [一句话描述]

- ...
```

提示：
> `📌 项目经历已生成（共 X 个项目）。回复"继续"生成技能，或指定修改某个项目。`

---

#### 4.5 生成技能列表

**写作指引**：
- 技能按 JD 重要性重新排序（JD 必须技能放前面）
- 分类清晰，不要堆砌
- 用 `` `熟练` `` `` `了解` `` `` `使用过` `` 标注掌握程度（可选）
- 去掉与 JD 完全不相关的技能

输出格式：
```markdown
## 专业技能

- **[核心技能类别]**：技能A `熟练` / 技能B / 技能C
- **[次要技能类别]**：技能D / 技能E
- **[工具/软件]**：工具A / 工具B
- **[软技能]**：能力A / 能力B（如相关）
```

提示：
> `📌 技能列表已生成。回复"继续"生成教育背景，或调整技能分类。`

---

#### 4.6 生成教育背景 + 证书

输出格式：
```markdown
## 教育背景

### [学校名称] | [专业] | [学历] | [YYYY.09] - [YYYY.06]

- [相关荣誉/GPA，如与JD相关]

## 证书 & 荣誉

- [证书名称] | [颁发机构] | [YYYY]（按JD相关性排序）
```

提示：
> `📌 教育背景已生成。回复"合并输出"生成完整简历，或修改此节。`

## 输出格式规范（react-resume-site 兼容）

以下是 mujicv.com 支持的特殊语法，生成时必须遵守：

### 容器语法

```markdown
::::: header      # 头部信息区
:::: title        # 标题容器
::: left          # 左侧列
::: right         # 右侧列
```

### 图标语法（联系方式区）

```markdown
[icon:phone 13800138000]
[icon:email xxx@gmail.com](mailto:xxx@gmail.com)
[icon:github GitHub](https://github.com/username)
[icon:blog 个人博客](https://example.com)
[icon:juejin 掘金](https://juejin.cn/user/xxx)
[icon:weixin 微信号]
```

### 技术标签

```markdown
### 项目名称 `React` `TypeScript` `Node.js`
```

### 标题层级规则

- `#` — 简历最顶部（姓名 - 职位）
- `##` — 主要板块（工作经历、项目经历、技能、教育背景）
- `###` — 子项目（每段工作/每个项目）

---

## 关键设计原则

1. **JD 关键词穿透**：分析 JD 后提取的关键词，必须自然分散到简介、工作经历、项目经历各节
2. **量化优先**：有数字的描述比无数字的描述说服力强 10 倍。没有数字的要尝试估算或换成定性的强成果描述
3. **动词驱动**：每条 bullet 必须以有力动词开头（主导、设计、优化、推动、搭建、实现）
4. **一库多用**：个人经历库 `~/.Codex/personal-profile.md` 是长期资产，每次针对不同 JD 从库中筛选不同内容
5. **渐进式生成**：每节生成后**必须等待用户确认**，不得跳过直接继续
6. **格式严格**：输出的 Markdown 必须严格遵循 react-resume-site 语法，不使用非标准语法

## 常见问题

**Q：个人经历库文件在哪里？**
A：`~/.Codex/personal-profile.md`，可以用任何文本编辑器直接编辑。

**Q：可以针对不同岗位保存多份简历吗？**
A：skill 本身不保存生成的简历，建议你自行复制 Markdown 保存到本地。经历库是通用的，每次针对不同 JD 生成不同简历。

**Q：生成的 Markdown 在 mujicv.com 排版不对？**
A：检查是否使用了正确的容器语法（`:::`），并确保标题层级正确（`#` 只用一次，`##` 用于主板块）。

**Q：如何让简历更短（控制在一页内）？**
A：在对话中说「帮我精简到一页」，skill 会建议删减哪些内容，或压缩 bullet point 数量。
