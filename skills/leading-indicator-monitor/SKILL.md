# 领先指标监控系统 (Leading Indicator Monitor)

这个 Skill 帮助你追踪公司/股票的**领先指标**——那些能够预测未来业绩变化的早期信号。

## 核心理念

```
输入（因）→ 时间传导 → 输出（果）
早期信号    →  滞后期   → 财务结果/股价
```

大多数投资者只看"果"（财报、股价），而忽略了"因"。这个工具帮你追踪"因"。

## 何时使用

- 每1-3天检查某个股票/公司的最新信号
- 在财报季前评估预期
- 识别趋势转折的早期迹象
- 寻找市场还未充分定价的信息

## 使用方法

### 基础用法

```
/leading-indicator-monitor NVDA
```

或使用公司名称：

```
/leading-indicator-monitor 英伟达
```

### 指定分析深度

```
/leading-indicator-monitor NVDA --depth=quick    # 快速扫描（5分钟）
/leading-indicator-monitor NVDA --depth=standard # 标准分析（默认）
/leading-indicator-monitor NVDA --depth=deep     # 深度分析（全面）
```

### 指定关注领域

```
/leading-indicator-monitor NVDA --focus=supply-chain  # 聚焦供应链
/leading-indicator-monitor NVDA --focus=hiring        # 聚焦招聘
/leading-indicator-monitor NVDA --focus=sentiment     # 聚焦情绪
```

### 对比模式

```
/leading-indicator-monitor NVDA vs AMD  # 对比两家公司的信号
/leading-indicator-monitor 黄金 vs 有色金属  # 对比大宗商品
```

对比模式会生成包含**交易信号对比**的报告，帮助你选择更优的投资标的。

### 导出 PDF 报告

```
/leading-indicator-monitor NVDA --export=pdf   # 分析后自动导出PDF
/leading-indicator-monitor NVDA --save         # 保存Markdown + PDF
```

报告将保存到 `~/Documents/leading-indicator-reports/` 目录，文件名格式：
- Markdown: `NVDA_2026-01-31_1630.md`
- PDF: `NVDA_2026-01-31_1630.pdf`

## 执行指令

当用户触发此 Skill 时，按以下流程执行：

### 第一步：解析目标

1. 识别股票代码或公司名称
2. 确认公司所属行业和产业链位置
3. 识别该公司的关键业务驱动因素

### 第二步：多维度数据采集

使用 `WebSearch` 工具并行搜索以下维度：

#### 2.1 供应链信号
```
搜索关键词：
- "[公司名] supply chain 2025"
- "[公司名] supplier capacity utilization"
- "[关键供应商] production capacity [公司名]"
```

对于科技公司，特别关注：
- 代工厂产能（如TSMC、三星）
- 关键零部件供应（如HBM、CoWoS）
- 上游设备商订单

#### 2.2 招聘动态
```
搜索关键词：
- "[公司名] hiring jobs LinkedIn 2025"
- "[公司名] job postings trend"
- "site:altindex.com [股票代码] job posts"
```

关注点：
- 岗位数量变化趋势
- 招聘部门分布（哪些部门在扩张？）
- 高管级别变动
- 从竞争对手挖人情况

#### 2.3 新闻与情绪
```
搜索关键词：
- "[公司名] news sentiment analysis"
- "[股票代码] analyst rating change 2025"
- "[公司名] [行业关键词] latest"
```

关注点：
- 情绪分数变化
- 负面新闻出现频率
- 分析师评级调整
- 管理层公开表态

#### 2.4 下游需求信号
```
搜索关键词：
- "[主要客户] capex spending 2025"
- "[公司名] order backlog demand"
- "[终端市场] growth forecast"
```

对于ToB公司，关注：
- 主要客户的资本支出计划
- 终端市场需求趋势
- 订单积压情况

#### 2.5 竞争格局
```
搜索关键词：
- "[公司名] vs [竞争对手] market share 2025"
- "[行业] competition analysis"
- "[竞争对手] new product announcement"
```

#### 2.6 另类数据
```
搜索关键词：
- "[公司名] alternative data signal"
- "[公司名] web traffic app downloads"
- "[公司名] github activity developer"
```

### 第三步：信号分析与评分

对每个维度的信号进行评估：

| 信号类型 | 评分标准 |
|----------|----------|
| 🟢 强烈看多 | 多个独立信号指向同一方向，与历史模式一致 |
| 🟡 中性/混合 | 信号相互矛盾或无明显变化 |
| 🔴 警示信号 | 出现与历史牛市前不同的负面模式 |
| ⚪ 数据不足 | 无法获取可靠数据 |

### 第三步B：生成交易信号

基于领先指标分析，生成具体的交易操作建议：

#### 3B.1 搜索价格数据
```
搜索关键词：
- "[股票代码] stock price today"
- "[股票代码] technical analysis support resistance"
- "[股票代码] analyst price target 2026"
- "[资产名] price forecast"
```

#### 3B.2 确定交易信号

**综合信号判定规则**：

| 综合信号 | 判定条件 |
|----------|----------|
| 📈 **强烈买入** | 4-5个维度🟢，无🔴，基本面+技术面共振 |
| 🟢 **买入** | 3-4个维度🟢，最多1个🟡 |
| 🟡 **持有/观望** | 信号混合，或接近阻力位 |
| 🟠 **减仓** | 2-3个维度🔴，或技术破位 |
| 🔴 **卖出** | 多维度🔴，趋势明确转空 |

#### 3B.3 价格操作建议框架

**对于股票**：
- **建仓区间**：基于技术支撑位 + 估值合理区间
- **止损位**：通常设在关键支撑下方3-5%或重要均线下方
- **目标价**：
  - TP1（短期）：最近阻力位或分析师保守目标
  - TP2（中期）：分析师中位数目标价
  - TP3（长期）：分析师最乐观目标或基于增长估算
- **仓位建议**：基于信号强度和风险收益比

**对于大宗商品**：
- **建仓区间**：基于历史支撑+成本曲线+季节性
- **止损位**：关键心理价位或生产成本线
- **目标价**：基于供需分析+机构预测
- **交易工具建议**：ETF、期货、相关股票等

#### 3B.4 风险收益比计算

```
风险收益比 = (目标价 - 当前价) / (当前价 - 止损价)

推荐标准：
- 风险收益比 ≥ 2:1 → 值得交易
- 风险收益比 ≥ 3:1 → 优质机会
- 风险收益比 < 1.5:1 → 不建议追高
```

## 注意事项

1. **这不是投资建议** - 仅供研究参考
2. **数据有滞后** - 网络信息可能不是最新
3. **相关性≠因果** - 需要人工判断逻辑是否成立
4. **避免过度交易** - 高频监控是为了识别趋势，不是鼓励频繁交易
5. **组合验证** - 单一指标不可靠，多维度交叉验证更重要
