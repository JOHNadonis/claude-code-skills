# 正确的非共识 - 内容生产线

> 理论基础：李普曼《公众舆论》(1922) —— 刻板印象是节省认知成本的工具。打破它时必须提供新框架，否则人们会回到旧认知。

## 定位

本 skill 解决「说什么」，不解决「怎么说」。
- **说什么** → 本 skill：选题挖掘 + 论证结构
- **怎么说** → `/dontbesilent-writer`：文风和表达

两个 skill 可串联：先用本 skill 生成内容骨架，再用 dontbesilent-writer 润色成稿。

---

## 两种模式

### 模式判定

| 用户输入 | 模式 |
|----------|------|
| 一个领域/话题（如「做生意」「自媒体涨粉」「读书」） | **话题模式** |
| 一本书名（或书的内容/截图/PDF） | **书籍模式** |
| 不确定 | 问用户 |

---

## 话题模式

### 输入
- **领域/话题**：必填
- **输出格式**：短视频口播稿（默认）/ 图文帖子
- **数量**：默认 5 条选题

### 执行流程

#### Step 1：挖掘刻板印象

在该领域中，找出大众持有的简化因果关系。

**识别规则** —— 刻板印象的三种常见模式：

| 模式 | 结构 | 示例 |
|------|------|------|
| 有 X 就能 Y | 充分条件谬误 | 有流量就能赚钱 |
| 做 X 就会 Y | 因果简化 | 涨粉快就能变现 |
| X 是 Y 的原因 | 单因归因 | 成功是因为努力 |

**共同特征**：单一因果，忽略条件。

对每个领域，批量列出 8-10 个刻板印象，格式：

```
1. [刻板印象原文] ← [模式类型]
   大众为什么信：[认知成因，1 句话]
```

#### Step 2：筛选 + 生成反例

从 8-10 个中筛选 5 个最有内容价值的（标准：反例强、新框架清晰、与卖书场景关联度高）。

对每个刻板印象，找到真实的反例：

```
刻板印象：涨粉快就能变现
反例：涨粉最快的博主往往不赚钱（泛流量 vs 精准流量）
```

反例要求：
- 真实可信，不是编造
- 最好是众所周知但没人往这个方向想的事实
- 能让人「愣一下」

#### Step 3：构建新框架

打破旧认知后，必须提供新的认知框架替代。

新框架的结构：
```
旧认知：[A 导致 B]
新框架：[B 取决于 C 和 D 的匹配度]
关键概念定义：
  - C 的定义：……
  - D 的定义：……
```

**关键**：新框架中如果出现模糊概念（如「流量质量」「匹配度」），必须给出明确定义。定义不清楚，内容就不成立。

#### Step 4：展开为完整内容

根据用户选择的输出格式，将骨架展开为完整内容。

## 书籍模式

### 输入
- **书名**（或书的内容摘要/截图/PDF）：必填
- **输出格式**：短视频口播稿（默认）/ 图文帖子
- **数量**：默认 3-5 条选题

### 执行流程

#### Step 1：提取书的核心论点

从书中提取 3-5 个核心论点，每个论点用一句话概括。

#### Step 2：反向映射刻板印象

对每个核心论点，反向推导：这个论点击碎了大众的哪个刻板印象？

```
书的论点：[论点]
对应的刻板印象：[大众的错误认知]
击碎方式：[书用什么证据/逻辑击碎的]
```

#### Step 3：构建内容骨架

将「书的论点」转化为「正确的非共识」四步结构：
1. 识别刻板印象（开头 hook）
2. 提供反例（书中的证据）
3. 给出新框架（书的核心观点）
4. 展开定义（把书的概念讲清楚）

#### Step 4：展开为完整内容

同话题模式 Step 4。

**书籍模式的额外要求**：
- 内容结尾自然引出书名，不要硬广
- 引书的方式：「关于这个问题，XXX 在《书名》里有一个说法……」或「这个框架来自《书名》」
- 让观众感到「这本书能帮我解决刚才那个困惑」

## 输出格式

### 短视频口播稿（默认）

```markdown
## 选题 N：[标题 - 用反常识观点做标题]

### Hook（前 3 秒）
[一句反常识的断言，让人想继续听]

### 刻板印象点破
[指出大众的错误认知是什么，为什么错]

### 反例
[给出让人「愣一下」的真实反例]

### 新框架
[提供替代认知，定义清楚每个关键概念]

### 收尾
[短句断言 / 回扣开头 / 引出书名（书籍模式）]

---
预估时长：X 分钟
核心金句：「……」
关联书籍：《书名》（书籍模式）
```

### 图文帖子

```markdown
## 选题 N：[标题]

### 封面文案
[1 句话，适合做封面大字]

### 正文
[800-1500 字，段落短，适合手机阅读]
[结构：hook → 破 → 立 → 展开 → 收]

### 标签建议
#标签1 #标签2 #标签3

---
关联书籍：《书名》（书籍模式）
```

## 质量自检

每条内容生成后，自动过以下检查：

- [ ] 刻板印象是否真的是「大多数人相信的简化因果」？（不是小众误解）
- [ ] 反例是否真实可信？（不是编的）
- [ ] 新框架是否比旧认知更有解释力？（不是为了反而反）
- [ ] 新框架中的关键概念是否都有明确定义？（模糊概念 = 内容不成立）
- [ ] Hook 能否在 3 秒内抓住注意力？
- [ ] 书籍模式：引书是否自然？（不是硬广）

## 串联 dontbesilent-writer

当用户需要从骨架到成稿时：

1. 本 skill 输出内容骨架（选题 + 四步结构）
2. 用户选择要展开的选题
3. 调用 `/dontbesilent-writer` 将骨架写成完整文案

提示用户：「骨架已生成，说"写第 N 条"我用 dontbesilent 风格展开成稿。」

---

## 批量模式

当用户说「批量」或一次要多条时：

1. 先输出所有选题的骨架（标题 + 四步结构摘要）
2. 让用户挑选要展开的
3. 逐条展开选中的

不要一次性展开所有内容——让用户先筛选，避免浪费 token。
