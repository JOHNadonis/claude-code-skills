# Reverse Account — 自媒体账号逆向工程

## 任务定义

给定一个账号（平台 + 账号名/链接），输出：
1. **账号画像** — 定位、人设、目标受众
2. **内容策略** — 选题逻辑、内容结构、爆款规律
3. **运营节奏** — 发布频率、时间窗口、互动策略
4. **增长轨迹** — 关键转折点、增长飞轮
5. **变现路径** — 商业化方式与时机
6. **可复制要素** — 提炼出可直接套用的方法论

## 执行流程

### Step 1：信息采集

用 `/browse` 或 `/search` 收集以下数据：

```
必采：
- 账号主页（粉丝数、简介、认证信息）
- 最近 20-30 篇内容（标题、封面、点赞/收藏/评论数）
- 历史爆款（互动量 top 5-10）
- 评论区高赞留言（了解受众痛点）

选采（有条件时）：
- 账号创建时间 / 早期内容（了解起步策略）
- 合作品牌 / 广告内容（变现路径）
- 账号主的其他平台（矩阵策略）
```

### Step 2：模式识别

按 [分析框架](references/analysis-framework.md) 逐维度提取规律：

- **选题公式**：高频词、话题类型、情绪触发点
- **内容结构**：开头钩子、中间逻辑、结尾 CTA
- **封面/标题规律**：视觉风格、标题句式、数字/情绪词使用
- **发布节奏**：频率、时间段、节假日策略
- **互动设计**：评论引导、话题标签、@策略

### Step 3：增长拆解

重建增长曲线（即使没有精确数据，也要推断关键节点）：

```
冷启动期 → 破圈节点 → 稳定增长期 → 变现期
   ↓            ↓            ↓           ↓
靠什么起步   什么内容爆了   复制了什么   怎么赚钱的
```

### Step 4：输出报告

按 [报告模板](references/report-template.md) 输出结构化分析。

## 注意事项

- 没有精确数据时，用「推断」标注，不要编造数字
- 重点在**规律提炼**，不是数据罗列
- 最后的「可复制方法论」是核心交付，必须具体可操作
- 如果账号有明显的「不可复制要素」（如创始人个人IP、资源壁垒），要明确指出
