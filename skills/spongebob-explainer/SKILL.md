# 海绵宝宝概念解说文案生成器

用比奇堡的故事，讲透任何商业/经济/社会概念。目标：让零基础的人看完一遍就懂，并且记住。

## 六段论结构模板（严格执行）

### ① 开场钩子（20-30字）
- 公式：`[悬念定语] + "这就是/这才是" + [概念的冲击性描述]`
- 目标：3秒内让人停下来
- 禁止：平铺直叙开头，禁止以"今天我们来讲"开头

**示例结构：**
```
这是比奇堡最[形容词]、也最[反差形容词]的[概念类型]。
```

### ② 场景建立（40-60字）
- 给一个比奇堡角色设定与概念强相关的真实困境
- 必须交代：主角、具体困境、为什么这个困境无法用常规方法解决
- 用数字量化困境（时间/金额/数量）

### ③ 解决方案 + 核心揭示（60-80字）
- 主角用"目标概念"的方式解决了问题
- **必须包含**一段简短对话（2-4句话），让概念在对话中自然显现
- 用 `"这就是[概念名]最核心的逻辑——[一句话定义]"` 完成揭示

### ④ 隐患/代价揭露（40-60字）
- 引入目标概念的风险、代价或反直觉之处
- **必须用对比句**：`"正常的[X]逻辑是……但[主角场景]里……"`
- **必须加入参考文案没有的知识点**（该概念的深层机制、历史数据或边缘案例）

### ⑤ 机制展开 / 续命操作（40-60字）
- 展示概念在现实中的具体运作方式（至少2个具体"招数"或子机制）
- 用编号或"第一招/第二招"结构
- 每招配一个比奇堡场景动作

### ⑥ 哲学升华（40-50字）
- 从比奇堡故事 → 抽象到普通人的现实处境
- 提炼一条**可以指导行动**的人生智慧（不是感慨，是行动指南）
- 结尾用强有力陈述句收尾，禁止用疑问句结尾

---

## 语言风格硬性规定

**必须做：**
- 每句话不超过15字
- 至少2处疑问句制造悬念
- 至少1处转折（"但是"/"可问题是"/"没想到"）
- 数字必须具体（不说"很多钱"，说"一百万"）
- 角色对话必须有（让场景活起来）

**禁止做：**
- 书面语、长定语从句
- 被动句（"被认为是"之类）
- 解释性废话（"接下来我们来看"）
- 照抄任何已知的海绵宝宝故事情节
- 与输入的参考文案有相同的故事走向

---

## 知识密度要求

每篇文案必须包含：
1. 概念的**核心定义**（用故事展示，不是直接说）
2. 概念的**运作机制**（至少2个具体步骤）
3. 概念的**隐藏代价**或**反直觉之处**（这是记忆锚点）
4. 概念**与普通人的关联**（升华段落）

---

## 生成流程

接收到 `/spongebob-explainer [概念]` 时：

**Step 1** — 内部分析（不输出）
- 该概念的核心机制是什么？
- 哪个比奇堡角色最适合作为主角？
- 什么比奇堡场景最能映射该概念的使用场景？
- 该概念最反直觉的一个知识点是什么？

**Step 2** — 按六段论生成文案

**Step 3** — 自检
- 字数在320-380字之间？
- 包含至少1段对话？
- 包含至少2处疑问句？
- 包含至少1个"反直觉"知识点？
- 故事情节与常见海绵宝宝梗不重合？

**Step 4** — 直接输出最终文案（不加说明，不加结构标注）
